# 三级发布工作流

## 流程图

```
飞书文档（创作与协作）
        ↓
  [导出为 Markdown]
        ↓
    GitHub 仓库
        ↓ 自动索引
  AI 搜索可引用（GEO优化）
        ↓
    知乎 / 公众号（人类流量）
        ↓
      私域转化
```

## 操作步骤

### 1. 创建 GitHub Token

1. 登录 GitHub
2. 点击右上角头像 → **Settings**
3. 左侧最下方 → **Developer settings**
4. **Personal access tokens** → **Fine-grained tokens**
5. 点击 **Generate new token**
6. 选择资源：
   - 选择 **All repositories** 或指定仓库
   - 权限选择：
     - **Contents** → **Read and write**
     - **Metadata** → **Read**（自动勾选）
7. 生成后复制 token（只显示一次）

### 2. 设置环境变量

在 PowerShell 或 Bash 中设置：

**Bash:**
```bash
export GITHUB_TOKEN="ghp_你的token"
export REPO_OWNER="你的用户名"
export REPO_NAME="haifeng-ai-articles"
```

**PowerShell:**
```powershell
$env:GITHUB_TOKEN = "ghp_你的token"
$env:REPO_OWNER = "你的用户名"
$env:REPO_NAME = "haifeng-ai-articles"
```

### 3. 创建仓库并初始化

```bash
python github_content_push.py --create-repo --init --owner 你的用户名 --repo haifeng-ai-articles
```

### 4. 推送单篇文章

```bash
python github_content_push.py --file "C:\Users\jt\WorkBuddy\Claw\图像质量与可控性_解决方案全分析.md"
```

### 5. 批量推送

```bash
for file in "C:\Users\jt\WorkBuddy\Claw"\*_分析.md; do
  python github_content_push.py --file "$file"
done
```

## 文件命名规范

```
articles/
├── 2026-07-04-图像质量与可控性解决方案.md
├── 2026-07-04-从LoRA到参考图生成.md
├── 2026-07-04-视频平台选择与优先级.md
└── README.md
```

命名规则：`<日期>-<文章标题>.md`

这样方便：
- 按时间排序
- 生成目录索引
- 追踪内容资产

## 与知乎/公众号的联动

GitHub 文章发布成功后：

1. 取 GitHub 文章的 Markdown 原文
2. 知乎专栏：发布完整文章（带标题、目录、图片链接）
3. 公众号：生成 KIWI 日记版本或精简版
