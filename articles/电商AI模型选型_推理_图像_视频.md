# 电商AI Agent 模型选型指南 — 推理 / 图像 / 视频

> 电商场景的 AI 工作流，本质上就三块：**想什么（推理）** → **画什么（图像）** → **动什么（视频）**。
> 下面按场景拆，每类的选型逻辑、成本、落地案例。

---

## 全景

```
电商AI Agent工作流

用户需求
   │
   ▼
┌─────────────┐
│ 推理模型      │ ← 选品分析、客服对话、商品描述、策略决策
│ (LLM)        │    场景：ChatBot / RAG / 数据分析 / Prompt编排
└──────┬──────┘
       │ 输出文本 / 指令
       ▼
┌─────────────┐
│ 图像生成模型  │ ← 商品主图、详情页、广告素材、模特换装
│ (Diffusion)  │    场景：AI模特 / 场景图 / 产品渲染
└──────┬──────┘
       │ 输出图片
       ▼
┌─────────────┐
│ 视频生成模型  │ ← 带货视频、产品展示、动态广告
│ (DiT/Wan)    │    场景：电商短视频 / 直播切片 / 动态展示
└─────────────┘
```

## 一、推理模型（LLM）— 电商的"大脑"

### 1.1 电商场景分布

| 场景 | 用在哪 | 对模型的要求 | 流量/天（中型电商） |
|------|--------|------------|-------------------|
| **智能客服** | 回答商品咨询/尺码推荐/退换货 | 低延迟、高并发、强知识库检索 | 5000-20000条对话 |
| **商品描述生成** | SKU标题/详情页/卖点提炼 | 中文好、懂电商话术 | 50-200条/天 |
| **数据分析** | 销售复盘/退货归因/用户画像 | 长上下文、结构化输出 | 10-50次/天 |
| **广告文案** | 千川/巨量引擎投放文案 | 创意强、懂消费心理学 | 20-100条/天 |
| **直播话术** | 直播间实时话术推荐 | 低延迟、强情感识别 | 持续流式 |

| **豆包2.1 Pro** | 字节 | 256K | ★★★★★ | ★★★★ | ★★★★★ | 0.8元/百万Token |

### 1.3 电商选型策略

```
场景一：智能客服（高并发、低延迟）
→ 推荐：DeepSeek-V3 或 豆包2.1 Pro
原因：国内服务、低延迟、Batch API便宜
不推荐：GPT-5.5（贵、延迟高、中文不够好）

场景二：商品描述生成（批量、质量要求中）
→ 推荐：DeepSeek-V3（Batch API，成本极低）
搭配：本地Qwen-7B做初稿，DeepSeek做润色

场景三：数据分析/运营策略（长上下文、重推理）
→ 推荐：豆包2.1 Pro（256K上下文，可一次丢进整月数据）
原因：256K能塞进30天的销售数据+退货记录一起分析

场景四：直播实时话术（低延迟、流式）
→ 推荐：DeepSeek-V3 流式版
原因：TTFT（首Token延迟）<200ms，满足直播间实时需求
```

### 1.4 成本对比（电商月100万Token用量）

| 模型 | 月成本 | 备注 |
|------|--------|------|
| DeepSeek-V3 Batch | **~5元** | 推荐首选 |

## 二、图像生成模型 — 电商的"脸面"

### 2.1 电商场景分布

| 场景 | 月产出量（中型电商） | 对模型的要求 |
|------|-------------------|------------|
| **商品主图** | 50-200张/月 | 服装细节准确、LOGO无变形、背景干净 |
| **模特换装** | 100-500张/月 | 面部一致性、服装版型准确 |
| **场景图（卧室/客厅/户外）** | 20-50张/月 | 光影自然、场景真实 |
| **广告素材** | 100-300张/月 | 视觉冲击力、文字嵌入干净 |
| **详情页配图** | 200-1000张/月 | 多角度、一致性高 |

### 2.2 主流图像模型对比（2026年6月）

| 模型 | 公司 | 分辨率 | 电商适配度 | 服装细节 | 成本 | 备注 |
|------|------|--------|-----------|---------|------|------|
| **Seedream 5.0 Pro** ⭐ | 字节 | 最高4K | ★★★★★ | ★★★★★ | 0.3-1元/张 | 电商首选，Seedance生态一体 |
| **FLUX.1 Pro** | Black Forest | 最高4K | ★★★★ | ★★★★★ | 0.5-2元/张 | 细节最强，无中国优化 |
| **Midjourney V7** | MJ | 最高2K | ★★★ | ★★★★ | 0.3-0.8元/张 | 审美强但服装细节一般 |
| **DALL-E 4** | OpenAI | 最高4K | ★★★ | ★★★ | 0.5-1.5元/张 | 中文文字嵌入差 |
| **SDXL + LoRA（本地）** | 开源 | 最高2K | ★★★★★ | ★★★★★ | 0.01-0.05元/张 | 本地跑，成本极低，需调优 |

### 2.3 电商选型策略

```
场景一：服装模特换装（LoRA + 批量出图）
→ 推荐：SDXL + LoRA（本地）
原因：
  - 本地RTX 5090D，24GB显存，跑FLUX LoRA绰绰有余
  - 单张成本0.02元 vs API 0.5元 = 25倍差距
  - 模型训练一次后无限复用
不推荐：MJ V7（不能精确控制服装版型）

场景二：商品主图/广告素材（高质量、少量）
→ 推荐：Seedream 5.0 Pro
原因：
  - 4K输出、中文文字干净
  - 配合Seedance视频生成统一画风
  - 每张0.3元，月100张=30元

场景三：详情页配图（大批量、一致性要求高）
→ 推荐：FLUX.1 Pro + ControlNet
原因：
  - ControlNet保证构图一致
  - 换产品只换Prompt不换构图
  - 批量API划算

场景四：快速试款（低成本验证）
→ 推荐：SDXL LoRA + ComfyUI
原因：
  - 0成本（本地跑）
  - 10分钟出30张
  - 不好的直接删，不心疼
```

### 2.4 典型的电商图像工作流（服装类）

```
Step 1: 训练LoRA模型
  用30-50张商品实拍图训练
  成本：1次训练 ≈ 2h × 0.5元/度 × 500W ≈ 1元（电费）
  产出：一个专属LoRA模型文件（约150MB）

Step 2: 批量出图（ComfyUI工作流）
  输入：SKU列表 + 场景模板
  输出：每个SKU 5张图（正/侧/背/细节/场景）
  速度：10秒/张，100SKU × 5张 ≈ 1.5h
  成本：100SKU × 5张 × 0.02元 ≈ 10元

Step 3: 筛选+后期
  - 自动筛：CLIP评分+面部检测（去掉崩图）
  - 人工选：80选20
  - 提亮/去底：批量自动化

成本总计：
  LoRA训练：1元（一次性）
  批量出图：10元/批次
  后期处理：0元（自动化）
  如果走Seedream API：100SKU × 5张 × 0.5元 = 250元
  本地跑节省：25倍
```

## 三、视频生成模型 — 电商的"动态"

### 3.1 电商场景分布

| 场景 | 月产出量 | 时长 | 对模型的要求 |
|------|---------|------|------------|
| **带货短视频** | 30-100条 | 15-70s | 商品展示自然、模特动作流畅 |
| **产品360°展示** | 20-50条 | 10-20s | 旋转平滑、细节不糊 |
| **直播切片** | 100-500条 | 5-30s | 高光时刻提取+AI重生成 |
| **动态广告** | 20-60条 | 6-15s | 视觉冲击力 |
| **品牌片/宣传片** | 5-15条 | 30-120s | 电影感、光影高级 |

### 3.2 主流视频生成模型对比（2026年6月）

| 模型 | 公司 | 最长时长 | 分辨率 | 电商适配度 | 成本 | 备注 |
|------|------|---------|--------|-----------|------|------|
| **Seedance 2.5** ⭐ | 字节 | **30s原生** | 4K | ★★★★★ | 3-8元/条 | 电商首选，7月初API开放 |
| **Kling 3.0** | 快手 | 15s | 1080P | ★★★★ | 2-5元/条 | 中文场景好，画质中等 |
| **Wan 2.1** | 阿里 | 15s | 1080P | ★★★ | 1-3元/条 | 性价比高，细节一般 |
| **Sora 2** | OpenAI | 60s | 4K | ★★★ | 10-30元/条 | 贵，无中文优化 |
| **Vidu 2.0** | 生数 | 15s | 1080P | ★★★★ | 2-6元/条 | 中国团队，人物一致性好 |
| **LibTV（工作流编排）** | 火山 | 任意 | 可组合 | ★★★★★ | 组合工具 | 整合Seedance/Kling等 |

### 3.3 电商选型策略

```
场景一：服装带货全片（70s左右）
→ 推荐：Seedance 2.5（原生30s） + LibTV编排
原因：
  - 原生30s直出，不用分段拼接
  - 无损局部编辑：换衣服不用重渲全片
  - 3D线框预演：分镜确定后再生成
  - 4K输出
不推荐：Sora（贵、中文差、没有电商优化）

场景二：批量换款出片（50-100条/月）
→ 推荐：Seedance + LibTV JSON批量导入
原因：
  - 你之前做的那套分镜JSON + 一键导入脚本
  - 换产品只换imageGenerationPrompt
  - 其他分镜参数复用
  - 月100条 × 5元 = 500元 vs 传统拍摄 = 50000元

场景三：直播切片/快剪（量大、短）
→ 推荐：Wan 2.1 或 Kling 3.0
原因：
  - 时长6-15s，对画质要求没那么高
  - 1-3元/条，成本极低
  - 日产出30-50条不心疼

场景四：品牌宣传片（少量、高质量）
→ 推荐：Seedance 2.5 + 后期
原因：
  - 30s原生 + 4K，质感够品牌调性
  - 可以用50份参考素材并行约束
  - 人物/产品/场景一次锁定
```

### 3.4 你的场景最适合的路线

```
你现在的工作流已经跑通的模式：

推理（DeepSeek-V3）
  ↓ 写脚本 + 分镜
图像（Seedream 5.0 / SDXL LoRA）
  ↓ 出参考图 + Prompt草图
视频（Seedance 2.5 / Kling 3.0）
  ↓ 出片
LibTV编排
  ↓ 组合成片
交付

改成换产品时的操作：
1. 换产品参考图
2. 改imageGenerationPrompt里的产品描述
3. 其他分镜参数不动
4. 一键重新出片
```
