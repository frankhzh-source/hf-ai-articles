# 电商客服聊天记录清洗 — 详细SOP

> 从千牛/抖音/京麦导出的原始客服对话，不能直接喂给AI。
> 需要经过6步清洗，才能变成Agent可用的训练/知识库数据。

---

## 一、原始数据长什么样（以千牛为例）

**导出格式**：CSV/Excel，包含以下字段：
- 会话ID、时间、买家ID、客服ID
- 消息内容（文本）
- 消息类型（文字/图片/表情/卡片）
- 订单关联信息（如有）

**典型的"脏"数据**：
```
买家: 在吗
客服: 在的亲~
买家: 这款有货吗[图片]
客服: 有的呢
买家: 多少钱
客服: [自动回复]欢迎光临...
买家: 我是说那个红色的
客服: 亲这款129元
买家: 能便宜吗
客服: [表情:玫瑰]亲可以申请优惠券...
买家: 我微信号xxx发你红包
客服: 亲平台不允许...
买家: ****（脏话）
```

## 二、清洗6步法

### Step 1: 去敏处理（隐私脱敏）

**清洗内容**：
- 手机号 → 替换为 `[PHONE]`
- 微信号/QQ号 → 替换为 `[WECHAT]` 或删除整句
- 收货地址 → 替换为 `[ADDRESS]`
- 真实姓名 → 替换为 `[NAME]`
- 订单号/物流单号 → 保留后4位，其余脱敏

**示例**：
```
清洗前："我电话13812345678，地址杭州市西湖区xxx"
清洗后："我电话[PHONE]，地址[ADDRESS]"
```

**工具/方法**：
- 正则表达式匹配：`1[3-9]\d{9}` 匹配手机号
- 地址库匹配：省市区街道关键词
- 人工抽检：10%样本人工复核

**标准**：脱敏率≥99%，误杀率≤1%

---

### Step 2: 去噪处理（剔除无效内容）

**需要删除的内容**：

| 噪音类型 | 示例 | 处理 |
|---------|------|------|
| 自动回复/欢迎语 | "欢迎光临本店..." | 删除 |
| 表情符号 | [玫瑰][微笑][图片] | 删除或替换为`[EMOJI]` |
| 系统卡片 | "订单已发货"自动通知 | 删除 |
| 测试对话 | 客服自己发的测试消息 | 删除 |
| 单字/无意义回复 | "嗯""哦""好的" | 根据上下文保留或删除 |
| 平台敏感词 | 被****屏蔽的内容 | 整条删除或标记 |
| 跨平台引流 | "加我微信""去闲鱼拍" | 删除或标记违规 |

**清洗标准**：
- 噪音占比目标：从原始30-50%降到≤10%
- 有效对话轮次：保留≥70%

---

### Step 3: 会话结构化（分意图打标签）

**把对话按业务意图分类**：

| 标签类别 | 具体标签 | 示例 |
|---------|---------|------|
| 售前咨询 | `询价` `库存` `尺码` `材质` `发货时效` | "这款有M码吗？"→`库存`+`尺码` |
| 售中跟进 | `改地址` `改尺码` `催发货` `加急` | "能帮我改地址吗"→`改地址` |
| 售后处理 | `退货` `换货` `退款` `质量投诉` `物流投诉` | "收到的衣服破了"→`质量投诉` |
| 特殊场景 | `投诉威胁` `恶意差评` `刷单` `广告` | "不给退款就差评"→`投诉威胁` |

**操作流程**：
1. 按会话ID聚合完整对话
2. 抽取"买家第一句话"作为意图触发点
3. 标注主要意图（单会话最多2个标签）
4. 质检：抽检10%，一致性≥90%

**输出格式**：
```json
{
  "session_id": "20240627_001",
  "intent": ["库存", "尺码"],
  "cleaned_dialog": [
    {"role": "buyer", "content": "这款有M码吗"},
    {"role": "seller", "content": "有的亲，M码库存充足"}
  ]
}
```

---

### Step 4: 问答对抽取（Q&A Pairing）

**目标**：把多轮对话，抽成"问题-答案"对，用于RAG知识库

**抽取规则**：

| 情况 | 处理方式 | 示例 |
|------|---------|------|
| 一问一答 | 直接抽取 | Q:有货吗？A:有的 |
| 追问场景 | 合并为一条Q | Q:这款有M码吗？L码呢？A:M有，L缺货 |
| 多轮澄清 | 保留完整上下文 | Q:多少钱→A:129→Q:能便宜吗→A:可领券 |
| 答非所问 | 人工标注，单独处理 | 标记为`bad_case` |

**质量标准**：
- 问答相关性：≥95%
- 答案完整性：单条A能独立回答Q
- 平均问答对/会话：3-5对

---

### Step 5: 知识库格式化

**把清洗后的数据，按知识库类型分类存储**：

| 知识库 | 数据来源 | 格式示例 |
|--------|---------|---------|
| FAQ库 | 高频问答对 | Q:发货时效？A:48小时内 |
| 商品知识库 | 商品属性问答 | SKU_001: {尺码表, 面料, 色差说明} |
| 话术库 | 客服优秀回复 | 议价场景: "亲可申请优惠券..." |
| 边界Case库 | 难处理的对话 | 投诉威胁: {对话原文, 处理建议} |

**向量化存储**（可选）：
- 使用Embedding模型（如text-embedding-3-small）
- 存入向量库（Pinecone/Qdrant/Milvus）
- 用于RAG检索

---

### Step 6: 质量校验与反馈

**6.1 自动校验**

| 校验项 | 方法 | 通过标准 |
|-------|------|---------|
| 脱敏检查 | 正则扫描残留手机号/地址 | 0残留 |
| 重复率 | 文本相似度>90%的去重 | 重复率≤5% |
| 空值率 | 检查空Q或空A | 空值率≤1% |
| 长度异常 | Q>200字或A>500字标记 | 异常率≤3% |

**6.2 人工抽检**

- 抽检比例：10%（最低100条）
- 抽检维度：脱敏完整性、意图准确性、QA相关性
- 一致性要求：≥90%（抽检人与标注人一致）
- 不合格处理：整批回滚重洗

## 三、清洗成本详解（500-2000元/月怎么算的）

### 按数据量估算

| 月对话量 | 清洗工时 | 人工成本 | 工具/外包费 | 合计 |
|---------|---------|---------|------------|------|
| <1000条 | 8-12h | 200-400元 | 50元 | 250-450元 |
| 1000-5000条 | 20-30h | 600-1200元 | 100元 | 700-1300元 |
| 5000-20000条 | 40-60h | 1200-2400元 | 200元 | 1400-2600元 |
| 20000+条 | 需专职 | 4000+元 | 500元 | 4500+元 |

### 成本构成

```
清洗成本 = 标注人力 × 时薪 + 质检人力 × 时薪 + 工具费 + 项目管理费

以月5000条对话为例：
- 初洗：20h × 30元 = 600元
- 质检：5h × 40元 = 200元
- 工具（标注平台）：100元
- 项目管理（沟通/验收）：200元
- 合计：1100元（落在500-2000区间）
```

## 五、常见问题

### Q1: 能不能用AI自动清洗？

**可以，但需要人机结合**：
- 去敏、去噪、格式化 → AI自动（准确率95%）
- 意图标注 → AI初标+人工复核
- 质检 → 必须人工抽检10%

**成本对比**：
- 纯人工：1000条/人天
- AI辅助：3000-5000条/人天
- 成本降60-70%，但质检不能省

### Q2: 清洗好的数据用在哪？

1. **训练数据**：给Agent做Few-shot示例
2. **知识库（RAG）**：向量检索，实时回答
3. **话术库**：客服培训，标准回复模板
4. **Bad Case分析**：找出Agent答错的地方

### Q3: 多久清洗一次？

| 数据类型 | 清洗频率 | 说明 |
|---------|---------|------|
| 历史对话 | 一次性（首次）+ 增量月清 | 首次洗全量，之后每月洗新增 |
| 商品数据 | 上架时即时清洗 | SKU更新即洗 |
| 活动数据 | 活动期间周清 | 大促对话量暴增，需增量清洗 |
