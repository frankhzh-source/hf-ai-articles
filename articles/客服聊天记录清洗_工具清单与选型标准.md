## 二、路线A：自己动手（Python 脚本方案）

**适合**：有技术人员、月对话量 < 5000 条、预算 < 1000 元

### 工具栈推荐

| 环节 | 工具/库 | 用途 | 成本 |
|------|---------|------|------|
| 去敏 | `re`（正则） + `phone` 库 | 脱敏手机号/地址/微信号 | 免费 |
| 去噪 | `emoji` 库 + `fasttext` | 识别表情/语种过滤 | 免费 |
| 打标 | `transformers` + 轻量BERT | 意图识别/情感分类 | GPU按需 |
| QA抽取 | 自写规则 + `spaCy` | 问答对抽取 | 免费 |
| 质检 | `scikit-learn` + `Cohen Kappa` | 人工抽检一致性计算 | 免费 |

### 典型工作流（复制即用）

```python
# 最小可用清洗脚本结构
import re, emoji, json

# 1. 去敏
def desensitize(text):
    text = re.sub(r'1[3-9]\d{9}', '[PHONE]', text)  # 手机号
    text = re.sub(r'wx[0-9a-z]{6,20}', '[WECHAT]', text)  # 微信号
    text = re.sub(r'[^\x00-\x7F]+', '', text)  # 非ASCII字符
    return text

# 2. 去噪
def denoise(text):
    text = emoji.replace_emoji(text, '')  # 去除表情
    text = re.sub(r'欢迎光临|亲.*可以帮您|自动回复', '', text)  # 欢迎语
    if len(text.strip()) < 3: return None  # 去短
    return text.strip()

# 3. 打标（规则版，轻量）
INTENT_RULES = {
    '库存': ['有货', '库存', '现货', '缺货'],
    '尺码': ['尺码', 'M码', 'L码', '多大', '身高', '体重'],
    '价格': ['多少钱', '价格', '优惠', '降价', '便宜'],
}
def tag_intent(text):
    tags = []
    for intent, keywords in INTENT_RULES.items():
        if any(kw in text for kw in keywords):
            tags.append(intent)
    return tags

# 跑批
def clean_pipeline(raw_file, out_file):
    with open(raw_file, 'r', encoding='utf-8') as f:
        data = json.load(f)
    cleaned = []
    for session in data:
        for msg in session['messages']:
            text = desensitize(msg['text'])
            text = denoise(text)
            if not text: continue
            cleaned.append({
                'session_id': session['id'],
                'text': text,
                'intent': tag_intent(text)
            })
    with open(out_file, 'w') as f:
        json.dump(cleaned, f, ensure_ascii=False)
```

### 优缺点
- ✅ 0 工具费，代码在你手里，完全可控
- ✅ 1000 条对话约 10 分钟跑完
- ❌ 需要懂 Python
- ❌ 规则精度有限（意图识别约 70-80%）
- ❌ 无法处理复杂语义（如"那个"指代不明）

## 三、路线B：半自动（AI 标注平台 + 辅助清洗）

**适合**：有少量技术人力、月对话量 5000-20000 条、预算 1000-5000 元/月

### 推荐平台

| 平台 | 定位 | 核心功能 | 定价 | 适合 |
|------|------|---------|------|------|
| **Label Studio** ⭐ | 开源标注平台 | 文本分类/序列标注/对话标注/多人在线 | 免费（社区版） | 有部署能力 |
| **Doccano** | 开源文本标注 | 文本分类/序列标注/Seq2Seq | 免费 | 轻量需求 |
| **阿里云PAI-iLabel** | 云端智能标注 | 预标注 + 人工修正 + 自动质检 | 约0.5-2元/条 | 阿里系 |
| **京东智臻链** | 标注平台 | 电商场景预标注模型 | 约0.3-1元/条 | 京东系电商 |
| **百度智能标注** | 云端服务 | 主动学习选样 + 人工标注 | 约0.5-2元/条 | 通用场景 |

### Label Studio 实战流程（推荐方案）

**Step 1: 安装（5分钟）**
```bash
pip install label-studio
label-studio start
```

**Step 2: 导入数据**
- 支持 CSV/JSON/JSONL 直接导入
- 自动按会话分组

**Step 3: 配置标注模板**
- 使用 "Conversational" 模板
- 配置意图标签（如 `库存/尺码/价格/退换货/投诉`）
- 配置实体标签（如 `产品名/尺码/价格/手机号`）

**Step 4: AI 预标注**
```python
# Label Studio 支持接入 ML 后端做预标注
# 用你的规则引擎或轻量模型先标一遍
# 人工只需要"纠错"而非"从头标"
```

**Step 5: 导出**
- 导出为 JSON/CSV/CoNLL
- 直接作为 Agent 训练数据

### 半自动方案的成本对比

| 月对话量 | 纯人工 | AI辅助+人工纠错 | 节省 |
|---------|--------|---------------|------|
| 3000条 | 600元 | 200元 | 67% |

**核心逻辑**：AI 先标一遍（准确率约 80%），人只改错的 20%，效率提升 4-5 倍。

## 五、选型决策标准

### 5.1 核心考量维度

| 维度 | 自己写脚本 | 半自动平台 | 全外包 |
|------|-----------|-----------|--------|
| **月数据量** | <5000条 | 5000-20000条 | >20000条 |
| **技术能力** | 有Python基础 | 有IT运维 | 无技术要求 |
| **月预算** | <500元 | 500-3000元 | 3000-20000元 |
| **数据安全** | 最高（完全本地） | 中等（可私有部署） | 低（数据出域） |
| **控制粒度** | 100% | 80% | 30% |
| **起步速度** | 1-3天 | 1天 | 3-7天 |

### 5.2 按电商规模选型

| 你的规模 | 推荐路线 | 具体方案 | 月成本估 |
|---------|---------|---------|---------|
| 🏪 **小店**（月GMV<30万，月对话<1000） | 自己动手 | Python脚本跑清洗+手动打标 | 0-200元 |
| 🏬 **成长型**（月GMV 30-300万，月对话1000-5000） | 自己动手 或 半自动 | Label Studio + 规则引擎 | 200-1500元 |
| 🏢 **成熟型**（月GMV 300-3000万，月对话5000-20000） | 半自动 或 外包 | 阿里PAI + 质检外包 | 1500-8000元 |
| 🏭 **品牌/头部**（月GMV>3000万，月对话>20000） | 全外包 + 自建兜底 | 幻想客服类 + 自建标注组 | 8000-50000元 |

### 5.3 关键选型决策树

```
Q1: 你有会编程的人吗？
  ├→ 没有 → Q3
  └→ 有 → Q2

Q2: 月对话量 < 5000 条？
  ├→ 是 → 自己写 Python 脚本（路线A）
  │  推荐: re + fasttext + 简单规则
  └→ 否 → 上半自动标注平台（路线B）
      推荐: Label Studio + ML 后端

Q3: 数据能出公司吗？
  ├→ 不能（数据安全要求高）→ 自己买工具私有部署
  │  推荐: Label Studio 本地部署
  └→ 能 → Q4

Q4: 月预算 > 3000 元？
  ├→ 是 → 找外包公司（路线C）
  │  推荐: 幻想客服（综合能力最强）
  └→ 否 → 用 SaaS 平台轻量方案
      推荐: 云帆未来 SaaS 中台 1980元起
```

### 5.4 选型避坑清单

| 常见坑 | 后果 | 怎么避 |
|-------|------|--------|
| 只看AI解决率 | 虚标，实际数据清洗不到位效果差很多 | 要求看到他们清洗前后的对比样本 |
| 只看Token单价 | L3严重低估，后期追加预算 | 要求对方给出L3的详细报价明细 |
| 数据出去了拿不回 | 被服务商锁死 | 合同中明确标注"清洗后数据归商家所有，可导出原始格式" |
| 外包公司用的不是你的真实数据 | 模型效果差 | 要求必须使用商家真实对话数据清洗，不能用通用语料 |
| 只洗一次不管后续 | 过3个月就不行了 | 合同中规定月度增量清洗频率 和 QA 抽检标准 |

## 六、工具选型的"最低可行方案"（MVP）

如果你今天就要开始，从0到1的推荐路径：

```
第1周：Python 脚本（正则去敏+去噪）
       成本：0元，产出：干净文本
       
第3周：加规则意图打标
       成本：0元，产出：带标签语料

第6周：起步用 Label Studio 做手动质检
       成本：免费，产出：质检报告

第12周：数据量上来了 → 升级到半自动或外包
        成本：按需，产出：成熟标注流水线
```

**核心原则**：不要一步到位买最贵的工具。先跑通最小闭环，按量扩。
