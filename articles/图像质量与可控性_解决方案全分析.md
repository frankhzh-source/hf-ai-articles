# 图像生成质量与可控性 · 全面解决方案分析

## 核心问题拆解

"保证质量 + 是自己想要的"——这两个目标其实是两件不同的事，解决方案不完全重叠：

```
质量 = 画质高低（清晰度/光影/纹理/无扭曲）
可控性 = 想什么来什么（构图/风格/角色/姿势/光影精确复现）
```

两者有时同向发展（更好的模型 = 更高画质 + 更好理解 prompt），有时冲突（为了精确控制牺牲画质）。

---

## 一、Prompt 层解决方案（门槛最低，效果上限也最低）

### 1.1 结构化 Prompt 工程

**原理：** 把自然语言的"模糊描述"拆解为机器可解析的结构化指令。

| 方法 | 做法 | 质量提升 | 可控性提升 |
|:----:|:----|:--------:|:---------:|
| **分段模板** | [主体]+[细节]+[环境]+[光照]+[构图]+[质量后缀] | 👍 | 👍 |
| **负面提示词** | 显式排除 bad quality, deformed, blurry | 👍👍 | — |
| **权重语法** | (keyword:1.3) 或 (keyword)++ 强调关键元素 | — | 👍 |
| **CFG Scale 调节** | 7.0=平衡 / 10.0=严格遵循prompt但画质下降 | 🟡 | 👍👍 |
| **种子固定** | 固定 seed 值，同一 prompt 产出可复现 | — | 👍👍 |

**适用场景：** 所有模型通用，零成本。
**局限：** 上限天花板明显——prompt 写得再好也弥补不了模型自身的理解局限。

### 1.2 质量后缀词库（你已在使用）

不同场景挂不同质量后缀，对 Final 画质有明显影响：

| 场景 | 推荐后缀 |
|:----:|---------|
| 电商主图 | 8K, highly detailed, sharp focus, soft studio lighting |
| 服装材质 | sharp fabric texture, clear print detail, 8K, macro |
| 人像/肤质 | natural skin texture, no smoothing, subtle details |

此方案你已经在用，投入产出比最高的 prompt 层优化。

## 二、模型选择层（根本性解决方案）

### 2.1 模型选型决定质量上限

| 模型 | 画质上限 | prompt遵循 | 角色一致性 |
|:----:|:-------:|:----------:|:---------:|
| **Seedream 4.6** | ★★★★★ | ★★★★ | ★★★★★ |
| **Flux Pro 2** | ★★★★★ | ★★★★ | ★★★★ |
| **DALL-E 4** | ★★★★ | ★★★★★ | ★★★ |
| **Midjourney V7** | ★★★★★ | ★★★ | ★★ |
| **SD 3.5 + Adapter** | ★★★★ | ★★★ | ★★★ |

**核心结论：** 质量天花板由模型决定，prompt 只是在这个天花板内优化。你选的 Seedream 4.6 已经是电商场景的天花板。

### 2.2 多模型组合策略（关键认知升级）

> **不要用同一个模型解决所有问题。**

| 任务 | 最佳模型 | 原因 |
|:----:|:--------:|:-----|
| 主视觉图（角色+服装一致） | Seedream 4.5/4.6 | 14张参考图，一致性最强 |
| 高质感产品特写 | Flux Pro / Seedream 4.6 | 纹理细节最真实 |
| 带文字的促销图 | Ideogram 3 | 文字准确率 95% |
| 快速试稿/概念探索 | Seedream 5.0 Lite | 速度快，质量够看 |
| 复杂场景精确描述 | DALL-E 4 | GPT-5 语义理解最强 |

**这是目前最务实的质量保证方案——不追求"一个模型打天下"，而是为每个任务选最合适的工具。**

## 三、参考图层（一致性保障的核心方案）

### 3.1 参考图的核心质量因子

参考图的质量直接影响输出图的最终质量。好参考图 = 好输出成立的前提是参考图本身够好。

| 因子 | 影响 | 最佳实践 |
|:----:|:----|:--------|
| **参考图分辨率** | 🔴 强 | 不低于 1024×1024，建议 2048×2048 |
| **光照一致性** | 🔴 强 | 多张参考图的光照方向尽量统一 |
| **背景纯净度** | 🟡 中 | 白底图/纯色背景效果最好，复杂背景会干扰主体特征提取 |
| **参考图张数** | 🟡 中 | 电商场景 3 张（白底图+平铺图+模特图）够了，超过 4 张可能引入冲突 |

### 3.2 参考图权重策略

| 权重值 | 效果 | 适用场景 |
|:-----:|:----|:--------|
| 0.0-0.3 | 几乎无视参考图 | ❌ 不适用 |
| 0.3-0.5 | 参考图为"建议性质" | 风格参考、氛围参考 |
| 0.5-0.7 | 参考图为"主要参考" | 面部特征保持一致 |
| **0.7-0.9** | **参考图强约束** | ✅ **电商服装/商品一致性** |
| 0.9-1.0 | 几乎复制参考图 | 不适合——可能直接复制原图姿势 |

**你的经验是对的：0.8-0.9 是电商角色一致性的甜蜜点。**

## 四、迭代层（从"接近"到"精确"的工作流）

### 4.1 多轮迭代路径

最实用的质量保证策略，不是"一次生成就完美"——是**在迭代中逼近**。

```
出图 → 检查 → 发现偏差 → 调整 prompt / 换参考图 / 改参数 → 再生成
         ↓ 如果反复调整都不行
       换模型 → 重新开始
         ↓
        如果还不行
        → LoRA 微调 / 图生图修改
```

### 4.2 图生图修改（Image-to-Image / Inpainting）

| 方法 | 原理 | 质量保证 | 适用场景 |
|:----:|:-----|:--------:|:--------|
| **局部重绘（Inpainting）** | 只修改选中区域，保留其他部分 | ✅ 精准修改 | 手部修复、面部微调、替换背景 |
| **整体图生图** | 基于原图结构和 prompt 重新生成 | 🟡 保留部分结构但可能整体变 | 风格迁移、大范围修改 |
| **ControlNet Tile** | 在不破坏原图结构的前提下增强细节 | ✅ 保持原构图+提升画质 | 低分辨率图增加细节、补充纹理 |

**Inpainting 是你当前最有用的工具**——生成的手崩了？遮住手重新生成一次，整个背景不用重做。

### 4.3 多轮生成 + 批量筛选

| 做法 | 质量保障逻辑 |
|:----|:------------|
| 同一 prompt 生成 4 张 | 模型每次跑有随机性，4 张里挑最好的 |
| 多组 seed 批量跑 | 固定 prompt 跑 10-20 张，用 seed 控制可复现 |
| 自动筛选工具（Aesthetic Scorer） | 用 CLIP Score / Aesthetic Score 自动打分排序 |

**实战建议：** 电商主图至少生成 4 张起挑。这不是浪费——重复生成到第 3-4 张时，出现完美结果的概率大幅提升。

## 五、后期增强层（最后的画质兜底）

### 5.1 超分辨率（Upscale）

| 工具 | 效果 | 速度 | 适用 |
|:----:|:----|:----|:-----|
| **模型内置 Upscale**（Seedream "增强"） | ✅✅ | 快 | 日常使用 |
| **独立 Upscaler**（ESRGAN / Real-ESRGAN） | ✅✅✅ | 中 | 需要极致细节的场景 |
| **AI 超分**（Topaz Gigapixel / Magnific AI） | ✅✅✅✅ | 慢 | 商用级精修 |

### 5.2 修复出图缺陷

| 缺陷 | 修复方案 |
|:----|:--------|
| 手指细节崩了 | Inpainting 局部重绘 / 扩展画布后补手 / 裁剪掉手 |
| 面部五官偏移 | 面部修复模型（GFPGAN / CodeFormer） |
| 纹理模糊 | 图生图 + ControlNet Tile + 更高 quality prompt |
| 背景杂乱 | 后期抠图替换背景（RemBg / SAM） |
| 画面噪点 | AI 降噪（跟 Upscaler 集成） |

### 5.3 伪影过滤清单（每次交付前自查）

```
□ 手指数量正确？关节正常？
□ 面部五官对称？无扭曲？
□ 服装印花文字清晰？没有乱码字符？
□ 布料褶皱自然？没有异常变形？
□ 光影一致？主光位置统一？
□ 背景没有"熔化"的物体？
□ 参考图和输出图的角色特征一致？
```

## 六、工程化层（批量生产的质量保障）

### 6.1 质量 Pipeline（你正在搭建的方向）

```
原始内容（参考图 + prompt）
  → 自动生成（Seedream / Flux）
    → 自动质量评分（Aesthetic Score + CLIP Score）
      → 合格 → 入库/发下游
      → 不合格 → 自动触发调整（降低 CFG / 换参考图 / 换 seed）→ 重试
```

核心逻辑：**不靠人每张盯，靠自动化筛选保证产出的最低质量线。**

### 6.2 A/B 测试 Workflow

当你遇到"两个方案不知道哪个更好"时：

```
方案A：参考图权重 0.8 + prompt 写法一
方案B：参考图权重 0.9 + prompt 写法二
    ↓
各生成 5 张 → 混排 → 盲选 → 记录胜出方案的参数
```

跑 3-5 次后，你就有了一套针对你的品类的"最优参数配方"。
