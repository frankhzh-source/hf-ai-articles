# 十维提示词精写 · 维度6：手部与姿态

## 为什么手部是AI生图的最大翻车点？

文生图模型（Seedream/SDXL）对手的理解远不如对脸的理解。原因是：
- **训练数据稀疏**：人像数据中面部标注密集，手部姿态标注稀疏
- **空间关系复杂**：手指间相对位置、遮挡关系、透视变形——模型没有"手部骨骼"概念
- **概率平均陷阱**：模型学到的"手"是所有角度手的概率平均，所以生成五根手指的平均值——就是糊在一起

## 六大翻车类型 & 根本原因

| 翻车现象 | 根因 | 解决方案 |
|---------|------|---------|
| 手指粘连 | 模型没分清手指间的边界 | 描述手势+手指分离状态 |
| 六指/四指 | 手指数量计数失败 | 强调"five fingers, clearly separated" |
| 手指错位 | 手部角度描述模糊 | 精确描述手掌朝向+手指方向 |
| 手腕变形 | 手和袖口衔接处丢失信息 | 描述袖口+手腕过渡 |
| 手部扭曲 | 透视角度+动作幅度过大 | 避免复杂透视角度 |
| 手和脸糊 | 手脸距离太近，模型混淆 | 保持手脸间距30px以上 |

## 一、手部Prompt精写公式

```
[手势描述] + [手掌朝向] + [手指状态] + [动作动词] + [负向提示]
```

### 手势描述 — 标准词库

| 手势 | ✅ 好写法（精确描述） | ❌ 坏写法（模糊） |
|------|---------------------|-----------------|
| 自然垂放 | 右手自然垂放于身侧，五指自然微屈，指尖朝下 | 手放旁边 |
| 轻抚面料 | 右手食指和拇指轻轻捏住衣领边缘，其余三指自然弯曲 | 摸衣服 |
| 展示手心 | 右手掌心朝向镜头，五指自然张开不并拢 | 手掌打开 |
| 握住物体 | 右手握拳，拇指在外，食指和拇指环绕杯柄 | 拿着杯子 |
| 叉腰 | 左手叉腰，虎口朝上，四指并拢贴于腰侧，拇指朝后 | 手叉腰 |
| 指向前方 | 右手食指伸直，其余四指握拳，手背朝上 | 指一下 |
| 整理头发 | 右手食指和中指轻轻夹住一缕头发，手肘抬起 | 撩头发 |
| 抱臂 | 双臂交叉抱于胸前，左手搭在右臂上，右手藏在左臂下 | 抱手 |
| 双手捧物 | 双手掌心朝上，手指微屈呈捧碗状，两拇指并拢 | 捧着 |
| 垂臂提物 | 右手自然下垂，手指微屈呈提握状 | 手上提东西 |

### 手掌朝向对照

| 描述 | 视觉表现 |
|------|---------|
| 手背朝向镜头 | 只能看到手背，看不到掌心和手指正面 |
| 掌心朝向镜头 | 能看到五个手指正面和手掌 |
| 掌心朝内 | 手指朝向镜头，掌心面向身体 |
| 掌心朝下 | 手水平，掌心向下 |
| 掌心朝上 | 手水平，掌心向上，手指微屈 |
| 手侧对镜头 | 拇指可见，仅食指侧面可见 |

### 手指状态精写

| 状态 | 好写法 |
|------|--------|
| 张开 | 五指自然张开，指间有明显空隙，每根手指轮廓清晰可见 |
| 并拢 | 四指并拢无间隙，拇指自然分开 |
| 微屈 | 手指自然弯曲呈弧形，从指根到指尖逐节弯曲 |
| 捏合 | 仅拇指和食指接触，其余三指自然弯曲握拳 |
| 握拳 | 四指握紧包住拇指，手指不可见，指节线在拳背清晰可见 |

## 二、电商家居服场景：标准手部动作库

### 场景1：展示服装（正面站立）

```
✅ 写法：双臂自然垂放于身体两侧，五指微屈自然放松，指尖朝下
         右手轻轻搭在腰侧的衣摆边缘，拇指在外可见，四指在衣料内侧
         左手自然下垂，手指朝向地面，手背朝向镜头
```

### 场景2：展示面料质感（近景/中近景）

```
✅ 写法：右手食指和拇指轻轻捏起袖口面料，让面料悬垂形成褶皱
         其余三指自然弯曲握起，手背朝上
         指尖轻微泛白，显示捏合力度恰好
```

### 场景3：展示印花（手部指向印花图案）

```
✅ 写法：右手食指指向胸前印花位置，其余四指握拳，手背朝向镜头
         手腕自然，手臂与身体呈45度角
         指尖距面料约2cm，不遮挡印花
```

### 场景4：手持衣架展示

```
✅ 写法：右手握住衣架挂钩处，食指在上拇指在下，其余三指握住横杆
         手部与衣架接触点清晰可见，指节分明
         衣架不遮挡面部
```

### 场景5：抚摸面料（细腻感）

```
✅ 写法：右手掌心轻贴在面料表面，五指自然张开
         手指轻轻划过面料产生微小的褶皱
         食指和中指略微前伸，无名指和小指稍后
         手部动作轻柔，指尖接触面料
```

### 场景6：转体动态

```
✅ 写法：身体转向右侧，左手自然垂放于身侧，右手沿身体侧面自然滑动
         双手手掌朝向身体，指尖自然朝下
         动态过程中手指不扭曲
```

### 场景7：插口袋

```
✅ 写法：右手插入侧边口袋，仅拇指露在口袋外
         其余四指在口袋内不可见，手背平滑无凸起
         左手自然垂放
```

### 场景8：整理领口

```
✅ 写法：双手食指和拇指捏住青果领两端的领角
         中指自然撑在领口两侧，其余手指微屈
         双手动作对称，腕部略微抬起
```

## 三、从参考图反推手部描述

当你有参考图但不知道怎么写时，按以下步骤拆解：

**Step 1：确定手的朝向**
→ 手背/掌心/侧对镜头？（从参考图观察）

**Step 2：确定手指状态**
→ 张开/并拢/微屈/捏合/握拳？

**Step 3：确定手和其他物体的关系**
→ 触摸/握住/捏起/指向/插袋/扶靠？

**Step 4：确定手部与身体的相对位置**
→ 腰侧/胸前/衣领/袖口/发梢/口袋？

**Step 5：写出完整描述**
→ [左手/右手/双手] + [朝向] + [手指状态] + [触摸/握持/指向] + [位置]

示例：
```
右手→手背朝向镜头→四指并拢拇指自然→指向印花标志→胸前
→ 右手抬起至胸前，手背朝向镜头
   拇指自然张开，其余四指并拢
   指尖指向胸前上衣口袋上方的刺绣标志
   距标志约3cm，不遮挡
```

## 四、反面案例库

| ❌ 用户怎么写 | 模型理解了什么 | 翻车结果 |
|--------------|---------------|---------|
| "手放桌上" | 手的位置→"桌上"，但手的形状→概率平均 | 一团肉色 |
| "拿着手机" | 手和手机的关系→"拿"，但手的具体形状→不明确 | 手和手机融合 |
| "手捧脸" | 手和脸很近→概率混合 | 手指和脸糊在一起 |
| "挥手" | "挥动"→动态模糊 | 手指拉丝 |
| "比个心" | 复杂的对称结构→概率平均 | 六根手指 |

| ✅ 应该这样写 | 模型理解了什么 | 生成结果 |
|-------------|---------------|---------|
| "右手掌心朝下平放在桌面上，五指自然张开，指间有缝隙" | 精确的手部位置+朝向+手指状态 | 清晰的手+五根分开的手指 |
| "右手握住手机，拇指在手机右侧屏幕边缘，其余四指在手机背面扣住" | 手指具体放在手机的哪个部位 | 真实握持感 |
| "右手轻搭在左脸颧骨下方，手指自然微屈不施力，指尖朝上" | 手部+脸部接触点的精确位置 | 不糊脸 |
| "右手向上抬起至肩高，掌心朝向镜头，五指张开" | 精确的抬手动作+五指分开 | 清晰五指 |
| "双手在胸前合拢，左手掌心贴右手手背，十指交叉紧扣" | 精确的双手相对位置 | 清晰的十指交叉 |

## 六、Seedream & Seedance 手部控制实操

### 图像节点（Seedream）

1. **每镜单独写手部描述**：不要用笼统的"hand"，每镜写清楚手在做什么
2. **参考图中手部清晰**：上传的主视觉肖像参考图中手部必须是清晰的——参考图本身手就糊，生成一定糊
3. **权重测试**：手部翻车时，尝试调整参考图权重0.75-0.85之间
4. **多图测试**：同一Prompt跑3轮，手部全翻车就改描述，2/3清晰就保留

### 视频节点（Seedance/Kling）

1. **手部运动要慢**：手部运动快的视频，手指必定糊。设duration≥5s
2. **表演强度≤0.4**：强度高→动作大→手部动态模糊
3. **减少手部交互**：如果手和其他物体交互翻车率高，改为手附近但不接触（隔2-3cm）
4. **小幅度动作更安全**：手的微动（手指轻颤/缓缓张开）比大幅度动作稳定得多

## 七、快速决策表

| 出片类型 | 手部风险 | 推荐策略 |
|---------|---------|---------|
| 主视觉肖像 | ⭐ 低（手不在画面） | 不用管手 |
| 全身站立 | ⭐⭐⭐ 中 | 用自然垂放手势，精确描述朝向+手指状态 |
| 中景展示服装 | ⭐⭐⭐⭐ 高 | 手部动作要简单（捏衣角/轻搭腰侧），避免手脸重叠 |
| 近景面料特写 | ⭐⭐⭐⭐⭐ 极高 | 手部动作幅度极小，用慢动作，负向词拉满 |
| 动态转体 | ⭐⭐⭐⭐ 高 | 手沿身体自然滑动，不要悬空 |
| 手持道具 | ⭐⭐⭐⭐⭐ 极高 | 道具形状要简单（衣架>手机>杯子），接触点清晰 |
| 双手动作 | ⭐⭐⭐⭐⭐⭐ 极限 | 两只手同时动作的翻车率是单手的3倍，尽可能改为单手 |

## 八、手部Prompt自检清单

写完后逐条检查：

- [ ] 指定了左手还是右手？
- [ ] 写了手掌朝向（手背/掌心/侧对）？
- [ ] 写了手指状态（张/合/屈/捏/握）？
- [ ] 手指数量明确（五指分开/四指并拢拇指分开）？
- [ ] 手和其他物体的接触点描述清楚？
- [ ] 手和脸/身体有没有重叠区域？
- [ ] 左右手动作有没有对称描述？（双手场景）
- [ ] 负向提示词里加了手部专用词？
- [ ] 参考图的手部是否清晰？
