# 维度 3 · 表情设定 — AIGC 专业区分版

## 核心认知（从业者必看）

**李一帆的公式（发生原因+表情细节+肢体动作+说话语气）是纯视频 Prompt 范式。** 它的核心价值在时间轴——让 AI 理解情绪的「渐变过程」。

但 LibTV 图像节点和 Seedance 视频节点对 Prompt 的要求完全不同，不能混用。

| 维度 | 🖼️ 图像节点（LibTV/Seedream） | 🎬 视频节点（Seedance/Kling） |
|------|------------------------------|------------------------------|
| **时间** | 无时间轴——一个凝固的瞬间 | 有时间轴——情绪的起始→变化→结果 |
| **原因** | ❌ 无用。静态图没法表现「因为看了什么」 | ✅ 必需。驱动整个情绪变化弧线 |
| **动作** | 描述「姿势」，不是动作过程 | 描述「动作过程」，有始有终 |
| **语气** | ❌ 无用。静图没声音 | ✅ 可呈现。口型、呼吸、微表情 |
| **表情颗粒度** | 精确到「嘴角上扬角度+眼神方向+眉头状态」 | 精确到「从X到Y的变化速度+幅度」 |
| **电商适用性** | 默认输出就是电商级表情（克制/自然） | 默认输出偏影视级（夸张/戏剧化），需刻意克制 |

## 第一部分：给图像节点（LibTV Image / Seedream）

### 核心原则

**图像节点的表情 = 一个凝固在最佳瞬间的状态。**

不要写过程（「先愣了一瞬→随后嘴角上扬」），模型不知道应该停在哪个阶段，最安全的选择就是面瘫。

**正确做法：直接给「拍下来的那一瞬间长什么样」。**

### 1.1 单帧表情公式

```
表情 = 面部三要素（眉+眼+口）+ 头部倾向 + 气场状态
```

**面部三要素必须同时给出，缺一不可**——缺任何一个，模型都会随机发挥：

| 要素 | ✅ 好写法 | ❌ 坏写法 |
|------|---------|---------|
| 眉毛 | 眉头舒展，眉峰自然 | 好看的眉毛 |
| 眼睛 | 目光平视镜头，微微含笑 | 有神的眼睛 |
| 嘴巴 | 嘴角微微上扬，嘴唇自然闭合 | 微笑 |
| 头部 | 微微侧向右侧约15度 | 头歪 |
| 气场 | 姿态放松，有一种刚起床还没完全清醒的慵懒感 | 状态好 |

### 1.2 4组基准表情（图像版）

#### 喜·微笑
```
眉峰舒展自然，目光平视柔和，眼底带着淡淡笑意
嘴角自然上扬约5度，嘴唇闭合或微张
头微微偏向一侧，下巴微收
整体气质：柔和温暖，松弛不紧张
```

#### 静·专注
```
眉间平坦无褶皱，双目自然睁合，视线微微下垂或平视
口唇自然闭合，唇线放松不僵硬
头正直，不前倾不后仰
整体气质：安静沉浸，没有表演感
```

#### 忧·轻愁
```
眉间有浅淡的纵向褶皱，双眉内侧微微靠拢
目光略低于平行线，上眼睑轻微下压
嘴角自然闭合但无上扬趋势，唇线拉平
头不自觉地略微低垂
整体气质：若有所思，不是难过——是心里有事在想
```

#### 疑·好奇
```
单侧眉峰略高于另一侧，双眼微微睁大
嘴唇微张但不露齿，仿佛想说什么还没说出口
头微微前探，转向目标方向
整体气质：被某件事或某个声音吸引了注意，不是惊吓
```

### 1.3 图像节点 Prompt 结构

```
[人物+服装固定块] + [面部三要素] + [头部倾向] + [气场状态] + [光照/构图]
```

**实战示例（达芬奇密码洛神玫瑰）：**

```
❌ 错误写法：
女生拉开窗帘看到阳光，很开心，嘴角慢慢上扬（→ 模型选了一个不上不下的状态，面瘫）

✅ 正确写法：
亚洲女性，28岁，穿着洛神玫瑰印花青果领家居服
眉峰舒展，目光柔和看向镜头，眼底带着自然的温暖笑意
嘴角微微上扬约5度，唇自然闭合
头部正直微偏右，姿态松弛
暖光3000K，45度右侧柔光，中景胸像
```

**关键差异**：去掉「原因」（拉开窗帘看到阳光）、去掉「过程」（慢慢上扬），只保留「结果状态」。

### 1.4 电商表情的「最佳瞬间」选择表

同一情绪下，选择哪个瞬间取决于图片用途：

| 图片用途 | 推荐瞬间 | 为什么 |
|---------|---------|-------|
| 主视觉肖像 | 表情处于稳定期（不是刚笑也不是快收） | 耐看、不累、适合反复看 |
| 产品平铺+人物 | 专注看向产品，嘴微抿 | 引导视线到产品 |
| 氛围场景 | 表情略松弛，眼神不直对镜头 | 营造氛围，不做销售感 |
| 细节特写 | 表情最小化，嘴巴轻闭 | 不抢产品风头 |
| 3张连拍系列 | 每张表情方向一致但强度不同 | 避免3张图看起来是复制粘贴 |

### 1.5 图像节点翻车 & 修法

| 翻车现象 | 根因 | 修法 |
|---------|------|------|
| 面无���情/面瘫 | 只写了情绪关键词没写三要素 | 写全「眉+眼+口」三个部件 |
| 表情夸张像在演戏 | 用了「开心」「大笑」等影视词 | 改用「嘴角微扬」「笑意含而不露」 |
| 每张生成都不一样 | 表情描述太模糊 | 参数化：嘴角上扬5度、微微侧头15度 |
| 嘴角上去了但眼睛没笑 | 只写了嘴巴没写眼睛 | 加「眼底含笑」「眼神柔和」 |
| 远景人也微笑了没用 | 相同Prompt用在所有景别 | 远景只写气场不写面部细节 |

## 第二部分：给视频节点（Seedance / Kling / 即梦）

### 核心原则

**视频的表情 = 一个完整的变化弧线。** 李一帆的公式在这完全适用。

### 2.1 视频表情公式

```
真实AI情绪 = 发生原因 + 表情细节 + 肢体动作 + 说话语气
```

**四个要素缺一不可**——少任何一个，视频角色的表情都会有「摆拍感」。

### 2.2 视频 Prompt 的时间轴写法

```
[起始状态] → [发生了什么] → [身体如何反应] → [表情变化过程] → [结果/最终状态]
```

**写法规则：**
- 用「→」「然后」「随后」「逐渐」等时间词连接
- 不要写省略号（模型不理解「...」表时间流逝）
- 每个阶段至少写 1-2 个具体动作
- 电商视频不要超过 3 秒的情绪铺垫（太拖）

### 2.3 4组基准表情（视频版）

#### 喜·微笑
```
女生拿起手机看了一条消息，看完后目光停在屏幕上
随后嘴角慢慢上扬，眼睛里有了笑意
她轻轻摇摇头，放下手机，脸上还留着刚才的笑意
```

#### 静·专注
```
女生穿着家居服坐在飘窗上，手里端着咖啡杯
目光落在窗外的远处，没有特定焦点
眨眼的频率很慢，呼吸平缓
整个人像凝固在午后的光线里
```

#### 忧·轻愁
```
女生翻着手机相册，手指停在一张旧照片上
她的动作慢了下来，指尖在屏幕上轻轻摩挲
眉头微微蹙了一下，随后又松开
她轻轻呼出一口气，锁上屏幕把手机放在了一边
```

#### 疑·好奇
```
女生在房间里整理东西，突然听到一个声音
她停下手中的动作，身体微微转向声音的方向
眉毛抬了一下，眼睛微微眯起像是在仔细分辨
等了两秒后，她轻声说了句「有人吗？」
```

### 2.4 Seedance 2.0 实操参数建议

| 参数 | 推荐值 | 说明 |
|------|-------|------|
| 人物表演强度 | 0.3-0.5（电商）/ 0.7-1.0（剧情） | 电商不要开太高，会变做作 |
| 微表情开启 | ✅ 开启 | 没有微表情，表情是跳变的 |
| Camera Motion | 固定或极慢推 | 不要摇移，会模糊表情 |
| 时长 | 4-7秒 | 太长表情会漂，太短情绪没展开 |

### 2.5 电商视频 vs 剧情视频的「表情克制」

| 对比 | 🎬 剧情/影视 | 🛒 电商家居服 |
|------|------------|-------------|
| 表情幅度 | 大——张嘴大笑/泪流满面 | 小——嘴角轻扬/眼底泛光 |
| 动作幅度 | 大——跳起来/捂脸大哭 | 小——放下手机/轻抚衣领 |
| 情绪发展 | 5-15秒铺垫，复杂情绪 | 2-4秒，简单直接 |
| 说话语气 | 完整台词 | 最多轻声感叹，或不说话 |
| 眼神方向 | 看向对手/自然移动 | 看向镜头（电商需要展示）or 看向产品 |
| 头部运动 | 头部配合大幅动作 | 微动，不超过30度 |

### 2.6 视频节点翻车 & 修法

| 翻车现象 | 根因 | 修法 |
|---------|------|------|
| 表情跳变（直接从一个表情跳到另一个） | 没写过渡过程 | 加「随后」「逐渐」「然后」等时间连接词 |
| 角色情绪和场景不对称 | 原因写得太抽象 | 写清楚「看见什么→什么反应」，越具体越好 |
| 全程一个表情不动 | 只给了静态描述 | 按时间轴写：起始→触发→变化→结果 |
| 动作做完了表情才跟上 | 动作和表情写在不同句且无顺序 | 写成同一序列：「先...然后...最后」 |
| 情绪太假像摆拍 | 缺乏微小的次级动作 | 加一个不经意的细节（摸衣角、呼口气、咽一下） |

## 第三部分：图片 vs 视频 — 快速参考表

### 同一情绪，两种写法

| 情绪 | 🖼️ 图像写法（凝固定格） | 🎬 视频写法（变化弧线） |
|------|------------------------|----------------------|
| 微笑 | 嘴角微扬，眼底含笑，头微侧 | 看到手机消息后先愣了一瞬，随即嘴角慢慢上扬 |
| 专注 | 眉眼舒展，目光集中在手中书本 | 目光在书页上来回移动，偶尔停下来思考 |
| 忧思 | 眉间轻蹙，目光低垂，嘴角放平 | 翻到一张旧照片，动作慢了下来，眉头慢慢蹙起 |
| 好奇 | 眉毛微扬，眼略睁大，唇微张 | 听到声音停下动作，侧耳细听，缓缓转头 |
| 温柔 | 眼神柔和，嘴角自然微扬，呼吸平稳 | 把散落的头发轻轻拢到耳后，看对方的眼神慢慢变得柔软 |

### 输出策略选择

```
做电商主图/详情页 → 用图像写法（静态，克制，直接输出最终状态）
做短视频/种草视频 → 用视频写法（有过程，有弧线，要有时间连接词）
做直播切片/口播 → 视频写法 + 开启微表情 + 表演强度0.4
做LibTV多图视频 → 图像写法 + 拆成关键帧放入不同帧节点
```

## 第四部分：从业者工具箱

### 4.1 图像节点的「表情固定块」（建议做进模板库）

```
眉[舒展/微蹙/微扬]，目[柔和/专注/低垂/微眯/含笑]，口[微扬/闭合/微张]
头[微侧左/微侧右/正直/微低]，气场[松弛/专注/慵懒/好奇/温暖]
```

每次出图时，从括号里选一组组合即可。

### 4.2 视频节点的「情绪模板库」

建议把以下 4 种常见电商短视频情绪预制成模板：

1. **拆快递惊喜**：撕开胶带→看到实物→愣住→嘴角上扬→手轻轻抚摸面料→满意地点头
2. **穿上的舒适感**：从衣架上取下衣服→穿上→低头看了一眼→对着镜子转了半圈→嘴角微扬
3. **被窗外吸引**：在窗边看书→被声音打断→抬头看向窗外→微微眯起眼睛→阳光照在脸上
4. **日常慵懒**：在沙发上调整了一下坐姿→闭上眼深呼吸→身体缓缓往后靠→放空两秒

### 4.3 LibTV 帧序列的「照片式表情」用法

LibTV 的「每张图 = 一个关键帧」，本质上还是图像生成。所以在 LibTV 里：

- ✅ 用图像版写法（抓取最佳瞬间的静态描述）
- ❌ 不要用视频版写法（模型会尝试猜时间轴，大概率猜错）

如果需要 LibTV 做有情绪变化的序列，正确做法是：

```
帧1（平静）→ 帧2（注意到什么）→ 帧3（反应浮现）→ 帧4（达到顶峰）
     ↑ 每帧独立写静态描述，帧与帧之间只改表情参数不变服装/构图
```

## 第五部分：这个框架的局限和下一步

李一帆的框架解决了「从抽象关键词到具体表演指令」的问题，但它不解决：

1. **角色一致性**：同个角色在不同镜头里表情变化时，脸不变——这是 维度1+维度4 的活儿
2. **服装/环境绑定**：表情变化时不走样——这是 维度5 的活儿
3. **多角色互动**：两个人的情绪互相影响——不在本框架范围内

对于 AIGC 从业者来说，正确的认知是：

> 表情控制不是独立技能。表情定了，剩下的还要确保「同张脸」「同件衣服」「同个灯光」——缺哪一个，这个表情就白生了。
