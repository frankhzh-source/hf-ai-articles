# AIGC 视频主角一致性保障框架

## 一、前置资产准备（出片前完成）

### 1.1 角色锚定资产

| 资产 | 用途 | 推荐做法 |
|------|------|---------|
| 正面半身照（1张） | 脸型基准 | 同一光线、同一角度，高分辨率 |
| 正面全身照（1张） | 全身比例+服装基准 | 站立自然姿态，手脚位置可见 |
| 侧身45°照（1张） | 侧面轮廓 | 左/右各一张更佳 |
| 纯侧面90°照（1张） | 侧影+发型厚度 | 确保耳朵、颧骨、下巴线清晰 |
| 背面照（1张） | 发型背面+服装背面 | 后颈线清晰，马尾/发髻高度可见 |
| 三/四组表情（喜、静、忧、疑） | 情绪稳定性 | 同一服装+背景，仅换表情 |
| 参考场景缩略图（N张） | 场景光照+空间 | 每个关键场景至少1张 |

### 1.2 道具/细节检查锚点

- **服装**：每件单品至少1张独立静物照（体现材质、纹理、褪色程度）
- **随身道具**：每个道具独立拍摄（体现磨损、位置、使用方式）
- **妆容/伤痕**：特写记录（确定位置、颜色、形状）
- **手表/首饰**：佩戴位置、朝向、是否反光

### 1.3 训练前置

- 优先做 LoRA/Reference（Seedance、可灵、Pika 均支持）
- 用1.1中的3-4张图训练，确保人脸-LoRA收敛
- 必须做一次"跨镜头一致性验证"：同一Prompt+不同Seed，检查脸型是否漂移

### 2.2 镜头过渡约束

| 过渡类型 | 一致性风险 | 应对 |
|---------|-----------|------|
| 跳景别（远→特） | 人物会重新生成，细节漂移 | 特写必须显式声明服装细节 |
| 跳时间（白天→夜间） | 光源色温变化 | 统一色温关键词 |
| 跳情绪（喜→悲） | 表情幅度过大 | 中景+过渡帧软切 |
| 插入新道具 | 道具凭空出现 | 前一个镜头先用手触碰该道具 |

## 三、分阶段质检流程

### 3.1 分镜预览阶段

1. 【Prompt一致性检查】
   - 全部15镜的主角描述块，逐字段比对
   - 确保服装、道具、发型在每行一致

2. 【参考图对齐检查】
   - 每个镜头确认参考图ID
   - 特写镜头必须单独配参考细节图（如：领口特写配领口参考）

### 3.2 图片阶段

3. 【单帧一致性抽检】
   - 每3个镜头抽检1帧
   - 对比基准帧（第1镜的第1帧）
   - 检查点：脸型、服装颜色、道具位置
   - 通过标准：四项达标≥3项

4. 【同场景多镜头并排检查】
   - 同一场景的全部镜头拼成网格
   - 逐行比对主角服装、色调、道具
   - 若2个以上镜头不一致，整场景重渲

### 3.3 视频阶段

5. 【连续帧抖动检查】
   - 播放逐帧（3帧/s），检查是否出现"跳脸"
   - 常见问题：鼻子突然变宽/眼睛颜色变化
   - 如有跳脸，标记前后各5帧，局部重渲

6. 【道具递推检查】
   - 道具在连续镜头中的位置是否合理
   - 如：录音笔在第1镜左手→第3镜右手→需确认中间有过渡动作

## 四、常用工具的实测能力

| 工具 | 一致性方案 | 实测效果 | 局限性 |
|------|-----------|---------|--------|
| Seedance 2.5 | Reference + Prompt | 服装/发型一致性较好 | 面部多角度不稳定 |
| 可灵AI 1.6 | 图生视频+角色参考 | 面部稳定（最新版本） | 道具容易丢失 |
| Pika 2.0 | Scene + Character ref | 场景一致性突出 | 长镜头会渐变漂移 |
| Sora | Prompt驱动 | 物理世界高度一致 | 对细节控制力弱 |
| Kling 1.6 | 图生视频+LoRA | 全身一致性较好 | 细节点（耳环等）容易丢失 |

---

## 五、常见失败模式与归因

| 失败现象 | 根因 | 修复策略 |
|---------|------|---------|
| 同场景不同色调 | 参考图光源不一致 | 统一场景参考图的拍摄光照 |
| 道具忽有忽无 | 分镜描述遗漏 | 用模板强制每镜写入道具字段 |
| 表情跳脱 | 情绪Prompt太模糊 | 用"嘴角微扬"替代"微笑" |
| 服装突然变色 | 颜色描述不够精确 | 用色号（#ED887C）替代"粉色" |
| 发型长度浮动 | 未标注发尾位置 | 标注"及肩胛骨下缘" |
