# L6 运维监控 — 电商场景成本细则拆解

> **运维监控是一层根本没人算、但缺少它 Agent 随时可能裸奔的成本。**
>
> 装监控，Agent 凌晨崩了你 2:01 收到飞书告警，5 分钟切备用模型。
> 不装监控，Agent 凌晨崩了你早上 9 点发现，中间 7 个小时客户咨询全部断联。

---

## 一、L6 到底在做什么？

把监控理解为给 Agent 做"体检"——持续检查它的健康、性能、精度、成本。

| 监控维度 | 体检类比 | 监控什么 |
|---------|---------|---------|
| **可用性** | 测体温 | Agent 是否在线、能否正常响应 |
| **性能** | 测心率 | 响应时间、Token 消耗 |
| **精度** | 检查报告 | 回答正确率、Bad Case 率 |
| **成本** | 查账单 | 日/周/月 Token 花费 |
| **安全** | 查血常规 | 异常请求、越权操作、提示注入 |

---

## 二、L6 成本公式

```
L6 = 监控基础设施 + 告警通道 + 人工值班 + 日志系统

其中：
- 监控基础设施 = 可用性探针 + 性能采集 + 精度评估 + 成本追踪
- 告警通道 = 飞书/钉钉/企微 Webhook + 电话/短信（严重告警）
- 人工值班 = 日常巡检 + 异常响应 + 复盘
- 日志系统 = 存储 + 检索 + 归档
```

---

## 三、电商场景各子项成本拆解

### 3.1 可用性监控（必须，成本极低）

做最简单的事：定时给 Agent 发请求，看它有没有正常返回。

| 方案 | 一次性 | 月成本 | 说明 |
|------|-------|--------|------|
| 自写脚本 + CRON | 0.5天（400元） | 0元 | 每 5 分钟发一次请求做健康检查 |
| UptimeRobot 免费版 | 0元 | 0元 | 5 分钟间隔，5 个监控点免费 |
| 服务器自建（Prometheus） | 2天（1600元） | 200-500元 | 专业级，但小型电商过剩 |

**电商推荐：UptimeRobot 免费版 + 自写 5 行脚本兜底，零成本。**

### 3.2 性能监控

监控三个关键数字：

| 指标 | 监控方式 | 成本 | 告警阈值 |
|------|---------|------|---------|
| Agent 响应时间 p95 | LLM API 响应时间埋点 | 200-500元/月 | >5s 告警 |
| 每日 Token 消耗 | 调用日志 SQL 统计 | 100-200元/月 | 日环比 >30% 告警 |
| API 调用错误率 | 捕获 API 异常返回码 | 100-200元/月 | >5% 告警 |
| 并发数 | 请求队列长度监控 | 0-100元/月 | 接近配额上限告警 |

**月成本合计：约 400-1000 元/月**

#### 关键指标的计算方式详解

**响应时间 p95 的计算：**
> 收集当天所有 Agent 响应的耗时，排序，取第 95% 分位的值。
> 简单实现：每条响应记一个 `{timestamp, latency_ms}` 到日志 → 每日定时 SQL 分析。
> 如果 p95 > 5 秒，说明模型响应太慢或 API 网关有瓶颈，需要优化。

**Token 消耗异常检测：**
> 记录每日的输入 Token / 输出 Token 总和。
> 用环比（今日 vs 昨日）来判断是否异常。
> 常见异常原因：Agent 陷入死循环（不停调工具）、某个客户发了超长对话、恶意攻击。

### 3.3 精度监控（最容易被忽略，但最重要）

Agent 每天回答几千个问题，怎么知道它答得对不对？

**方案：自动化抽样 + LLM-as-Judge 评估**

```
精度监控成本 = 抽样率 × 日对话量 × 评估成本

举例：
- 日对话量：5000 条
- 抽样率：5%（日抽 250 条）
- 评估方式：用 DeepSeek-V3 Batch API 评估每条回答质量
- 评估成本：每条约 0.01 元（输入约 1000 Token，输出约 200 Token）
- 日成本：250 × 0.01 = 2.5 元
- 月成本：2.5 × 30 = 75 元
```

| 精度维度 | 评估方式 | 评分标准 | 通过线 |
|---------|---------|---------|-------|
| **回答准确率** | LLM-as-Judge | 0-10 分 | ≥ 8.5 分 |
| **格式合规率** | 正则校验 | 通过/不通过 | ≥ 99% |
| **工具调用准确率** | 日志核对 | 工具选对+参数对 | ≥ 95% |
| **漏接率** | 应该转人工但没转的比例 | < 5% | ≤ 5% |
| **误判率** | 不该转人工但转了 | < 3% | ≤ 3% |

**额外成本：**

| 项目 | 月成本 | 说明 |
|------|--------|------|
| 自动化评估（LLM-as-Judge） | 75-150元 | 5% 抽样，DeepSeek-V3 Batch |
| 人工复核低分项 | 300-600元 | 评分 < 7 分的由运营审核，约 1 人天 |
| 周/月报告生成 | 200-400元 | 自动化产出，少量人工整理 |
| **月小计** | **575-1150元** | — |

### 3.4 日志系统

所有 Agent 的响应日志必须存储，否则出了事没法回溯。

| 方案 | 月成本 | 说明 |
|------|--------|------|
| 文件日志 + 按日轮转 | 0元 | 小规模够用，但检索困难 |
| PostgreSQL 存日志 | 200-500元 | 可 SQL 检索，适合中型 |
| Elasticsearch | 1000-3000元 | 专业全文检索，中小型过剩 |
| 云服务（阿里云SLS/腾讯云CLS） | 300-800元 | 按量计费，免运维 |

**电商推荐：PostgreSQL 方案。**

原因：
- 日 5000 条对话 × 平均 2KB = 10MB/天，PG 完全没压力
- 可以用 SQL 直接查 "昨天哪些客户说 '我要退货' 但 Agent 没正确引导"
- 月约 **300-500 元**（云 PG 实例多出来的存储和查询费用）

**日志的存储策略：**
- 热数据（近7天）：全量保留，快速检索
- 温数据（8-90天）：压缩存储，按需检索
- 冷数据（91天+）：归档到对象存储（阿里云OSS/腾讯云COS），月约 50-100 元
- **月总日志成本：约 350-600 元**

### 3.5 告警通道

| 通道 | 月成本 | 用途 |
|------|--------|------|
| 飞书群 Webhook | 0元 | 常规告警（精度下降/响应变慢） |
| 邮件 | 0元 | 日报/周报 |
| 电话/短信（备用） | 50-100元 | 严重告警（Agent 完全下线） |

**月成本：约 50-100 元**

### 3.6 人工值班

| 频率 | 耗时 | 做什么 |
|------|------|--------|
| 每日 | 15分钟 | 看昨日监控大盘：Token消耗/响应时间/精度分数 |
| 每周 | 1小时 | 复盘本周 Bad Case Top-10，确认根因 |
| 每月 | 2小时 | 月报 + 下月监控阈值调整 |

**月折算：约 0.5 人天 = 400 元**

## 四、完整案例：中型电商月 L6 成本

**假设：** 日对话 5000 条，1 个 Agent，单平台

| 子项 | 方案 | 月成本 |
|------|------|--------|
| 可用性监控 | UptimeRobot 免费 + 自脚本 | **0 元** |
| 性能监控 | 自写埋点 + SQL 分析 | **500 元** |
| 精度监控 | LLM-as-Judge 5% 抽样 | **750 元** |
| 日志系统 | PostgreSQL | **500 元** |
| 告警通道 | 飞书 Webhook + 短信备用 | **80 元** |
| 人工值班 | 日均 15 分钟 | **400 元** |
| **月合计** | — | **2,230 元/月** |

**年合计：2,230 × 12 = 26,760 元**

绝对值不高——关键是**大部分企业这 2,230 元一分没花，Agent 靠信仰运维。**

## 六、关键提醒

**你没有监控的时候，Agent 其实是裸奔的。**

常见的企业没装监控的状态：

| 情况 | 实际后果 | 监控后能做什么 |
|------|---------|-------------|
| Agent 精度从 90% 掉到 60% | 一上午给了几千个客户错误答案，发现时已晚 | 精度降到 80% 就自动告警，即时修复 |
| LLM API 故障 15 分钟 | 断了 15 分钟 → 几百条客户咨询无人回复 | 自动切到备用模型，客户无感知 |
| 某个 Prompt 改了导致回答风格突变 | 运营发现"今天 Agent 说话怪怪的" | 精度监控在 10 分钟内就会发现并告警 |
| Token 消耗因死循环暴增 10 倍 | 日调用费从 200 元涨到 2000 元，月底才发现 | 小时级 Token 监控，2 小时内定位异常 |

**一句话：L6 不是额外成本。不花监控的钱，花的都是事后补救的钱——而且绝对比监控贵得多。**
