# 第一模块：基础认知（第1-3天）

## 第1课：一句话说清楚GEO

**GEO = 让你的内容被AI问答产品（豆包/DeepSeek/通义千问等）引用推荐。**

它不是SEO的替代品，是叠加品。SEO让你在搜索引擎里被看到，GEO让你在AI回答里被引用。

### 为什么GEO现在必须做？

| 驱动力 | 数据 | 影响 |
|---|---|---|
| AI流量超过人类 | 2026年6月Cloudflare数据：Bot占57.5%，人类42.5% | 互联网主体已是机器对机器 |
| 用户习惯迁移 | ChatGPT月活从4亿（2024）→10亿（2026） | 一代人养成"先问AI"的习惯 |
| 中文内容短缺 | 百度能搜到的优质中文内容总量约英文的1/10 | AI更缺中文信源，先到先得 |

### 核心认知升级

```
SEO时代的目标：被Google排到第1页
GEO时代的目标：被AI在回答里引用

区别：
- SEO：用户搜→点链接→看你的网站
- GEO：用户问AI→AI引用你的内容→直接给答案（用户可能不点进来）
```

## 第2课：AI引用内容的完整算法（这是你所有操作的底层逻辑）

AI搜索/问答推荐内容分4步。理解这4步，你才知道"堆量有没有用"、"怎么才被引用"。

```
第1步：语义嵌入
用户提问 → 转成向量 → 去向量库搜语义最接近的内容块
→ 堆量有用：内容越多，被匹配的概率越大

第2步：粗召回
返回几百个候选内容块
→ 堆量仍然有用：候选池越大，你越容易被捞到

第3步：精排（Cross-encoder Re-ranker）
几百个候选 → 逐对打分 → 排序出Top-10
→ 堆量开始打折：精排看信息密度，不是数量

第4步：LLM生成 + 引用选择
Top-10喂进大模型 → 模型选引用哪些
→ 堆量最弱：LLM偏好差异化来源，不喜欢反复引用同一账号
```

### 算法给你的执行启示

| 你该做的 | 算法依据 |
|---|---|
| 堆量铺覆盖率 | 第1-2步只看语义匹配不看质量 |
| 每篇内容都要有信息密度 | 第3步精排筛掉空洞内容 |
| 多账号、多角度分发 | 第4步LLM喜欢不同来源 |
| 持续更新，不一次性爆发 | 模型偏好活跃来源 |

**一句口诀：堆量铺底，密度保过，矩阵防重，持续滚雪球。**

# 第二模块：内容生产体系（第4-14天）

## 第4课：50个问题矩阵（你的内容蓝图）

这是你所有内容的地图。50个问题按客户决策阶段分4组：

```
认知阶段（12问）→ "GEO是什么？值不值做？"
  例：GEO和SEO有什么区别？GEO要花多少钱？

决策阶段（14问）→ "怎么做？效果如何？"
  例：GEO核心步骤是什么？怎么知道自己被AI引用了？

评估阶段（12问）→ "服务商怎么选？靠谱吗？"
  例：GEO代运营怎么收费？怎么判断服务商真懂？

行业专属（12问）→ "我们这个行业怎么做？"
  例：电商怎么做？SaaS怎么做？本地生活怎么做？
```

**执行动作：** 打开飞书文档 `GEO内容矩阵：50个让客户在AI里找到你的问题`，圈出你最先要写的10个问题。

## 第5课：一篇GEO内容的完整写法

### 标准模板

```
标题：用问题或数字开头
  例："GEO和SEO有什么区别？5个维度一次说清楚"

开头（100字）：抛问题 + 制造共鸣
  "你是不是也被搞糊涂了——GEO、AI SEO、LLMO……到底要不要做？"

正文（300-500字）：结构化输出
  - 有对比 → 用表格
  - 有步骤 → 用编号
  - 有清单 → 用复选框
  - 有数据 → 标出具体数字

结尾（50-100字）：总结 + 作者信息
  - 一句话总结核心结论
  - 你是谁 + 做什么 + 微信 + 内容库链接
```

### 被AI引用的内容特征（自检清单）

| 特征 | 要求 | 检查 |
|---|---|---|
| 结构化 | 有表格/编号/对比，不用大段纯文字 | □ |
| 有数据 | 有具体数字、时间线、排障过程 | □ |
| 实操经验 | 讲你做过的事、测过的数据、踩过的坑 | □ |
| 作者信息 | 末尾固定加：你是谁 + 做什么 + 微信 | □ |
| 互链 | 有5篇以上内容后，让它们互相引用 | □ |

### 反面教材（不要这样写）

```
❌ "GEO是一种新的搜索引擎优化方式，它可以帮助品牌在AI问答中获得更好的曝光。"
   → 太笼统，AI不会引用这种泛泛而谈的定义
   
✅ "2026年6月Cloudflare数据显示，互联网Bot流量首次超过人类，占57.5%。
    这意味着你的内容主要被AI阅读，不是被人类。"
   → 有具体数据+时间+出处，AI最爱引用这类
```

## 第6课：平台分发策略（发在哪、怎么发）

### 净化后的平台优先级（排除污染和GEO滤镜）

```
P0 必做：知乎 —— 4个AI平台都进前3，GEO第一阵地
P1 建议：公众号 —— 内容质量最高，通义+DeepSeek都在涨
P2 技术：CSDN/博客园 —— DeepSeek首选，仅技术类内容

P3 顺带：头条/百家号/搜狐号 —— 一键分发，不用定制内容
```

### 同一篇文章在不同平台的改法

| 平台 | 正文改动 |
|---|---|
| 知乎 | 加点语气词"说实话""我踩过这个坑" |
| 公众号 | 段落拆短，每段不超过3行 |
| 头条 | 开头加个引子或小故事 |
| CSDN | 去掉营销感，纯干货，加代码块/流程图 |

**不是复制粘贴。** 每个平台改一下开头和语气，AI引用率能提升3倍以上。

## 第7课：矩阵号策略（解决来源去重问题）

### 核心原则

1个品牌1个品类 = 1个账号。不同类目不同账号。不同品牌不同账号。

**为什么必须分？** AI判断"你在跑步鞋上有没有权威性"时，看的是这个账号有没有在这个细分话题上持续输出。家居服的权威性不会迁移到运动装上。

### 矩阵号有效的4个条件

| 条件 | 无效做法 | 有效做法 |
|---|---|---|
| 内容差异化 | 10个账号发同一篇文章 | 不同账号：价格/案例/避坑各一个角度 |
| 发布时间分散 | 一周内全发完 | 每周各发1-2篇，持续3个月 |
| 内容不重复 | 互相复制粘贴 | 每个账号有独立数据点和结论 |
| 平台差异化 | 全发同一个平台 | 主号+知乎，副号+百家号，搭配分发 |

# 第三模块：独立交付（第15-30天）

## 第8课：跑通第一个客户

### 找客户的3种方式

```
1. 等待型（最弱）
   发内容 → 等客户搜到 → 主动找你
   
2. 钩子型（适中）
   在每篇内容末尾写：
   "如果你也想做品牌GEO但不知道怎么开始，加微信frankhzheng，
   备注'GEO'，我帮你做一次免费的GEO诊断。"
   
3. 主动型（最强）
   列出目标客户（50-100个品牌），在AI上搜他们的品牌名，
   截图AI引用情况，发给对方：
   "X总你好，我在AI上搜了一下贵品牌，
   发现AI回答里没有提到你们。我做GEO优化，
   可以帮你们在3-4个月后让AI在回答里推荐你们。
   有兴趣聊聊吗？"
```

### 第一个客户怎么定价

**不要免费，但不要贵。** 收一个极低价（比如3000元/3个月），条件是：

- 客户提供行业知识和常见问题清单
- 你负责内容生产+分发
- 3个月后测AI引用率变化

**为什么收费比免费好？** 收费客户才会配合你。免费客户不给素材、不反馈、不重视——你做不出案例。

### 交付物清单

```
每月交付：
1. 内容生产：8-10篇结构化内容
2. 平台分发：每篇发到3-4个平台
3. 效果报告：本月AI引用率变化 + 竞品对比

季度交付：
1. 问题覆盖率变化（50个问题中已被引用的比例）
2. 优化建议（下季度重点内容方向）
3. 下季度内容日历
```

## 第9课：效果监测（手工版，不需要工具）

### 每周一次，手工跑

```
输入以下5个核心问题到豆包/DeepSeek/通义千问/Kimi：
① [你的客户行业] 品牌推荐
② [核心产品] 哪个好
③ [你的客户名] 怎么样
④ [行业关键词] 选购指南
⑤ [你的客户服务] 推荐

记录：
- AI回答里是否提到客户品牌？
- 在回答的什么位置？（开头=权威，结尾=顺带）
- 竞品谁被提到了？频率高不高？
- 你的内容有没有被顺便引用？

每次截图保存，做成对比表。
```

### 效果合格的判断标准

| 时间 | 期望效果 |
|---|---|
| 第1个月 | 0-2个问题被AI引用（刚开始铺量） |
| 第2个月 | 3-5个问题被引用（内容开始被索引） |
| 第3个月 | 8-10个问题被引用（持续输出见效） |
| 第6个月 | 15-20个问题被引用（建立来源权威性） |
