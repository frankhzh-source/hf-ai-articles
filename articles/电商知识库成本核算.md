# 电商 AI Agent 知识库成本核算

> 清洗成本近乎归零了，那知识库呢？——知识库的账比清洗复杂，因为它是**持续构建 + 持续维护**的，而且选型路线会影响整体架构成本。

---

## 一、电商知识库有哪些

一个给 AI Agent 用的电商知识库，通常包含 5 个库：

| 知识库 | 内容 | 数据量参考（中型电商） | 更新频率 |
|--------|------|----------------------|---------|
| **商品库** | SKU属性/卖点/规格/价格/库存 | 500-2000条 | 每日（新SKU+调价） |
| **FAQ库** | 高频问答对 | 200-500条 | 每周（新增场景） |
| **话术库** | 场景化话术模板 | 100-300条 | 每月 |
| **政策库** | 退换货/物流/售后规则 | 30-50条 | 季度（政策变更） |
| **历史对话库** | 清洗后的QA对，用于RAG检索 | 5000-50000条 | 持续 |

---

## 二、知识库的三大成本项

```
总成本 = 内容构建成本 + 基础设施成本 + 维护运营成本
```

### 2.1 内容构建成本

LLM 之前：每一条都要人工写 → 贵
LLM 之后：LLM 从已有数据中抽取 + 人工审核 → 极低

**对比：**

| 知识库 | 传统构建方式 | 成本 | LLM构建方式 | 成本 |
|--------|------------|------|------------|------|
| 商品库 | 运营逐条填写SKU属性 | 5000-15000元 | 从ERP导出 → LLM结构化 → 人工审 | 500-2000元 |
| FAQ库 | 客服整理高频问答 | 3000-8000元 | 从历史对话抽QA → LLM去重归并 | 200-800元 |
| 话术库 | 资深客服撰写 | 5000-10000元 | 从优秀对话抽→LLM提炼→人工定稿 | 500-1500元 |
| 政策库 | 法务/运营整理 | 2000-5000元 | 文档直接喂给LLM结构化 | 500-1000元 |

> **用 LLM 做知识库构建，一次性成本降 70-80%。**

### 2.2 基础设施成本

这是知识库最核心的"硬成本"，取决于你选哪种架构。

#### 方案A：纯向量检索（主流）

```
用户提问 → 向量化(Embedding) → 向量库检索 → 返回Top-K → LLM回答
```

| 组件 | 选项 | 月成本 |
|------|------|--------|
| **Embedding 模型** | | |
| | OpenAI text-embedding-3-small | $0.02/百万Token |
| | 本地部署 bge-large-zh-v1.5（免费） | 0元（需GPU） |
| | 本地部署 nomic-embed-text（Ollama） | 0元（CPU可跑） |
| **向量数据库** | | |
| | Pinecone Free | 0元（100K向量封顶） |
| | Pinecone Standard（1 Pod） | ≈$70-150/月（≈500-1000元） |
| | Qdrant Cloud Free | 0元（1GB封顶） |
| | 自建Qdrant（Docker） | 服务器≈200-500元/月 |
| | Milvus自建 | 服务器≈500-1000元/月 |
| | Chroma（本地文件存储） | 0元 |
| **存储** | 向量+元数据存储 | 100-500元/月 |

#### 方案B：纯 LLM 上下文注入（零基础设施）

```
用户提问 → 在 prompt 里直接追加知识库内容 → LLM 回答
```

适用于知识库规模较小（<500条）的场景：

```python
# 把知识库塞进 system prompt
system_prompt = f"""
你是一个电商客服AI助手。以下是商品信息：
{json.dumps(product_info, ensure_ascii=False)}

以下是FAQ：
{json.dumps(faq_list, ensure_ascii=False)}

以下是退换货政策：
{policy_text}
"""

# 月成本：只有 Token 费用
# 假设每条对话上下文中嵌入 2000 Token 的知识库内容
# 月对话 10000 条 × 2000 Token × 输出500 Token
# = 2000万Token输入 + 500万Token输出
# 用 DeepSeek-V3 Batch: 约 20元/月
```

#### 方案C：向量检索 + LLM（标准RAG，推荐）

```
用户提问 → 向量化 → 检索Top-5 → 拼进prompt → LLM回答
```

**典型月成本分解（中型电商）：**

| 项目 | 规格 | 月成本 |
|------|------|--------|
| Embedding | 20000条知识 × 月30次更新 = 60万Token | ≈0.1元 |
| 在线Embedding | 月10000询 × 每次1个 → 10000次 × ≈0.00002元/次 | ≈0.2元 |
| 向量库 | Qdrant Cloud Free（1GB内免费） | 0元 |
| LLM检索响应 | 月10000次检索 × (2000知识+500输出)Token | ≈10元 |
| **月合计** | | **≈10-30元** |

### 2.3 维护运营成本

| 维护工作 | 频率 | 传统方式 | LLM方式 |
|---------|------|---------|---------|
| 新SKU入库 | 每日 | 运营手动写SKU → 5-10分钟/条 | LLM从商品标题自动生成 → 30秒/条 |
| FAQ更新 | 每周 | 客服整理新增问题 → 2-4h/周 | LLM从新增对话中自动检测高频问题 → 10min/周 |
| 知识库去重 | 每月 | 人工扫描 → 2-4h/月 | LLM自动检测相似度去重 → 5min/月 |
| 知识库质量检查 | 每月 | 抽检+客服反馈 → 4-8h/月 | LLM全量扫描 → 10min/月 |
| **月维护工时** | | **30-60h** | **3-6h** |
| **折合成本** | | **3000-6000元/月** | **300-800元/月** |

## 三、三档方案的总成本对比

假设：中型电商，500SKU + 300FAQ + 50政策 + 5000历史对话

### 方案A：零基础设施（全走 LLM prompt）

```
架构：知识库内容定期拼接进 system prompt
优点：0 基础设施，不用向量库
缺点：知识库规模受限，量大不行
```

| 成本项 | 一次性 | 月均 |
|--------|-------|------|
| 内容构建（LLM） | 2000元 | — |
| 基础设施 | 0元 | 0元 |
| LLM Token消耗 | — | 10000对话 × 2500Token × Batch价 ≈ 25元 |
| 维护（抽检+更新） | — | 300元 |
| **合计** | **2000元** | **325元/月** |
| **首年合计** | **2000 + 325×12 = 5,900元** | |

### 方案B：标准RAG（多退少补，推荐）

```
架构：Qdrant Cloud Free + 本地Embedding + Batch API
```

| 成本项 | 一次性 | 月均 |
|--------|-------|------|
| 内容构建（LLM） | 2000元 | — |
| Embedding（本地bge，CPU） | 0元 | 0元 |
| 向量库（Qdrant Free） | 0元 | 0元 |
| LLM查询费用 | — | 10000×2500Token×Batch价 ≈ 25元 |
| 维护 | — | 400元（含每周增量更新） |
| **合计** | **2000元** | **425元/月** |
| **首年合计** | **2000 + 425×12 = 7,100元** | |

### 方案C：商业化RAG（适合大品牌）

```
架构：Pinecone Standard + OpenAI Embedding + GPT-4o
```

| 成本项 | 一次性 | 月均 |
|--------|-------|------|
| OpenAI Embedding | — | 60万Token ≈ 0.02元 |
| Pinecone Pod | — | $150 ≈ 1050元 |
| GPT-4o 查询 | — | 10000条×(2000+500)Token×$2.5/百万输出 ≈ 62.5元 |
| 维护 | — | 500元 |
| **合计** | **2000元** | **~1612元/月** |
| **首年合计** | **2000 + 1612×12 = 21,344元** | |

### 三档对比

| | 方案A（纯prompt） | 方案B（自建RAG） | 方案C（商业化） |
|-----|:---:|:---:|:---:|
| 首年总成本 | **5,900元** | **7,100元** | **21,344元** |
| 适用知识库规模 | <500条 | 500-50000条 | 5000+条 |
| 技术门槛 | 极低 | 低 | 低 |
| 检索精度 | 一般（全量） | 高（向量检索） | 高（商业化） |
| 可扩展性 | 差 | 好 | 最好 |

## 四、知识库成本中"看不见的坑"

### 坑1：Embedding 不花钱，但重算很贵

每次更新知识库，要重新 Embedding 全量或者增量。月均更新30次听起来不多，但：
```
如果每次全量重算 20000条 × 每次≈500Token × 30次 = 3亿Token/月
用OpenAI Embedding: 3亿×$0.02/百万 = $6 = 42元
用本地模型: 0元
```
**解法**：增量更新（只 Embedding 新增部分）+ 用本地 Embedding 模型

### 坑2：RAG 检索质量不跟成本成正比

```
Qdrant Free + bge-small-zh（本地）= 0元，但检索精度照样能到 90%+
Pinecone + OpenAI Embedding = 贵，但不一定比前者好多少
```
**解法**：先跑通免费方案，不够用再升级。不要上来就上 Pinecone。

### 坑3：知识库的"维护衰减"

```
一个月不更新知识库 → 召回率下降 5-10%
三个月不更新 → 召回率下降 20-30%
Agent 开始回答"这个商品已下架"但其实还在卖
```
**解法**：知识库维护不是一次性工作，必须纳入日常运营的预算。

## 五、实际成本数字（你可能关心的）

**一个服装电商，月对话5000条，200SKU，跑标准RAG：**

```
┌─────────────────────────────────────────┐
│ 知识库初期搭建（一次性）                   │
├─────────────────────────────────────────┤
│ LLM生成SKU描述（200条）         600元      │
│ LLM抽取FAQ（从10000条历史对话）  300元      │
│ 政策文档结构化                     200元      │
│ 小计                           1,100元     │
├─────────────────────────────────────────┤
│ 每月持续运营                           │
├─────────────────────────────────────────┤
│ 新SKU入库（月均30条新增）        LLM自跑     │
│ FAQ更新（每周增量）              LLM自跑     │
│ 向量库存储（Qdrant Free）          0元      │
│ Embedding（本地bge）               0元      │
│ LLM查询Token                     20元      │
│ 人工抽检维护                        300元      │
│ 小计                            320元/月    │
├─────────────────────────────────────────┤
│ 首年总计  1,100 + 320×12 = 4,940元       │
└─────────────────────────────────────────┘
```

**核心结论：知识库的真正成本不在基础设施，在持续维护的人力。**

- 向量库：用 Qdrant Free / Chroma 可以做到 0 元
- Embedding：用本地模型可以做到 0 元
- LLM 查询：用 Batch API 月均 20-30 元
- **维护人力：300-800 元/月**（这才是大头，但比传统方式省了 5-10 倍）
