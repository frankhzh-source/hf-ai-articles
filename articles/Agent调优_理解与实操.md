# Agent 调优 — 理解与实操

> Agent 调优 ≠ 模型微调。模型微调改的是权重，Agent 调优改的是 **Prompt + 工具 + 检索 + 记忆 + 护栏** 五件套。

---

## 一、Agent 调优到底在调什么

一个电商 Agent 的完整链路：

```
用户提问
  ↓
① 意图识别（这是什么类型的请求？）
  ↓
② 工具选择（需要查库存？查物流？调优惠券？）
  ↓
③ 知识库检索（从 FAQ/商品库找相关信息）
  ↓
④ 生成回复（根据检索结果 + 上下文写回答）
  ↓
⑤ 格式校验（输出是否符合规范？有没有违规？）
  ↓
用户收到回复
```

**任何一个环节出问题，Agent 表现就崩。调优就是逐环节定位问题→改配置→验证→上线。**

## 二、五维调优框架

### 维度 1：Prompt 调优 — 最频繁的操作

**问题表现：** 回答太啰嗦/太简短/风格不对/不该答的答了

**调优方法：**

| 问题 | 症状 | 怎么调 | 示例 |
|------|------|--------|------|
| 角色不明确 | 回答像通用AI，不像客服 | 加角色定义 + 品牌风格 | 从"你是一个AI助手"→"你是XX品牌服装客服，语气亲切但不啰嗦" |
| 指令模糊 | 有时回有时不回 | 明确边界 | "只回答与商品相关的问题，不回答非商品问题" |
| 格式没约束 | 输出结构不统一 | 加输出格式 | "回复控制在50字以内，先回答再补充" |
| 缺乏边界 | 回答政治/价格对比等敏感问题 | 加护栏 | "不回答价格对比问题，引导用户查看商品详情页" |
| 工具调用不准 | 该查库存时不查 | 明确工具调用条件 | "当用户问有没有货时，必须先调用check_stock()函数" |

**电商实战案例：**

```
V1 Prompt："你是一个电商客服"
→ 回答太笼统，像机器人 ❌

V2："你是XX品牌服装客服，语气亲切"
→ 风格有了，但尺码推荐不准确 ❌

V3："推荐尺码时，先问身高体重，再对照尺码表"
→ 准确了，但回复太长 ❌

V4："推荐尺码时先给出建议码，再补充一句原因"
→ 简洁准确，通过 ✅

操作：修改 Prompt → 跑测试集 → 对比精度 → 上线
耗时：30分钟
```

### 维度 2：工具调优 — 该调的不调，不该调的瞎调

**问题表现：** Agent 调用错误的工具 / 参数传错 / 工具返回结果不会用

**调优方法：**

| 问题 | 原因 | 怎么调 |
|------|------|--------|
| 选了错误的工具 | 工具描述不清晰，模型没理解工具用途 | 重写工具描述（name + description 要精准） |
| 参数传错 | 参数名/格式与工具要求不匹配 | 检查工具函数的参数签名，调整描述 |
| 工具返回结果没用上 | 模型忽略了工具返回的数据 | 在 Prompt 中强调"必须根据工具返回结果来回答" |
| 该调工具时不调 | 模型自己猜答案，没走工具 | 降低 Prompt 中"你直接回答"的倾向，加"优先调用工具" |

**工具描述的写法差异：**

```
❌ 差：
def check_stock(sku_id):
    """查库存"""

✅ 好：
def check_stock(sku_id: str) -> dict:
    """
    查询指定SKU的实时库存信息。
    当用户问'有货吗''还有吗''什么时候补货'等问题时调用。
    返回: {"sku_id": "xxx", "available": true/false, "quantity": 100, "next_restock": "2026-07-01"}
    注意：调用后必须根据返回的quantity来回答用户。
    """
```

### 维度 3：检索调优 — RAG 质量的核心

**问题表现：** 答非所问 / 知识库内容找不到 / 找到不相关的

**调优方法：**

| 问题 | 原因 | 怎么调 |
|------|------|--------|
| 检索结果不相关 | Chunk 太大或太小 | 调整文本分割大小（500-1000字符实验） |
| 检索不到 | Embedding 模型不合适中文 | 换 bge-large-zh 或 text-embedding-3-small |
| 召回了但没用上 | 上下文窗口没传进去 | 减少检索条数（Top-K从10→3），只传最相关的 |
| 召回杂音多 | 知识库里有过时/重复内容 | 定期清理知识库，去重+过期删除 |

**电商 RAG 调优实战：**

```
问题："这个裙子有白色的吗？"

检索结果（调优前 Top-10）：
  1. 白色连衣裙 2024款（过季） ❌
  2. 黑色连衣裙 2025款 ❌
  3. 退换货政策 ❌
  4. 白色半身裙（不同品类） ❌
  ...（全是杂音）

调整方案：
  - 分割大小：1000→500字符（更精准）
  - Top-K：10→3（减少杂音）
  - 加元数据过滤：只查当前季商品

检索结果（调优后 Top-3）：
  1. 2025夏季连衣裙·白色·有货 ✅
  2. 2025夏季连衣裙·白色·M码库存5件 ✅
  3. 2025夏季连衣裙·白色·面料成分 ✅
```

### 维度 4：记忆调优 — 别让 Agent 失忆

**问题表现：** 用户刚才说了尺码，下一句又忘了 / 跨天对话丢失上下文

**调优方法：**

| 策略 | 适用场景 | 成本 | 实现方式 |
|------|---------|------|---------|
| 短期记忆（窗口） | 单次会话内 | 0 | 保留最近 N 轮对话 |
| 摘要记忆 | 长对话（>10轮） | Token成本 | 每几轮自动摘要历史 |
| 持久化记忆 | 跨会话 | 存储成本 | 存数据库（用户ID→偏好） |
| 关键信息提取 | 只记住重要的 | 适中 | 提取尺码/地址/订单号等实体 |

**电商场景的"关键信息记忆"：**

```
用户A第一次对话：
  "我身高165，体重110，穿什么码？"
  → Agent回答"M码"
  → 记忆模块提取：{用户ID: A, 尺码: "M", 身高: 165, 体重: 110}

用户A第二天又来：
  "上次说的那条白裙子有货吗？"
  → Agent查到历史："您上次问的是M码，这条裙子M码有货"
  → 用户不用重复说尺码 ✅
```

### 维度 5：护栏调优 — 不该答的别答

**问题表现：** Agent 回答了不该回答的问题（价格对比/竞品评价/违规承诺）

**调优方法：**

| 护栏类型 | 拦截什么 | 怎么配 |
|---------|---------|--------|
| 内容过滤 | 政治/色情/暴力 | 敏感词库 + LLM-as-Judge 检测 |
| 业务规则 | 虚假优惠/价格承诺 | 规则引擎 + 工具调用限制 |
| 输出校验 | JSON 格式/长度/合规 | Output Parser + 正则 + 二次校验 |
| 人工兜底 | Agent不确定时 | 置信度低于阈值→转人工 |

**电商特有的护栏：**

```
❌ Agent 不能说的：
  - "这件衣服是最好的/最便宜的"（广告法）
  - "明天一定到货"（物流承诺）
  - "我给您打个折"（无权限）
  - "您去别家看看吧"（引导流失）

✅ Agent 应该说的：
  - "这款目前有满200减30的活动"
  - "预计发货后2-3天到达"
  - "建议您联系在线客服申请优惠券"
  - "这款还有其他颜色可选"
```

## 三、调优的操作流程

### 3.1 日常调优 Cycle（30分钟/次）

```
发现问题（运营反馈/质检报告）
  ↓
定位环节（Prompt/工具/检索/记忆/护栏？）
  ↓
改配置（改Prompt/调工具描述/改检索参数）
  ↓
跑测试集（至少50条覆盖场景+边界）
  ↓
对比精度（改前 vs 改后）
  ↓
上线观察（灰度30%→全量）
  ↓
记录版本（改了啥/为什么改/效果如何）
```

### 3.2 调优频率建议

| 频率 | 做什么 | 产出 |
|------|--------|------|
| **每日**（10分钟） | 看 Bad Case | 标记当天异常回复 |
| **每周**（1小时） | 跑测试回归 + 修 Top-5 Bad Case | 精度提升 1-2% |
| **每月**（2小时） | 检索质量审计 + 知识库更新 | 召回率稳定 |
| **每季**（半天） | 大促版本适配 + 全量回归测试 | 大促稳定运行 |

### 3.3 电商 Agent 调优 Checklist

```
每日检查清单：
□ 查看质检报告，有没有异常回复
□ 标记当天 Top-3 Bad Case
□ 如果是工具调用错误→重写工具描述
□ 如果是检索没命中→检查知识库
□ 如果是风格问题→微调 Prompt

每周检查清单：
□ 跑标准测试集（100条），计算精度
□ 精度从上周的多少到多少？
□ 如果下降→回退到上周版本，排查变更
□ 如果上升→确认原因，固化配置
□ 更新 Changelog

每月检查清单：
□ 知识库质量全量扫描
□ 过期内容清理
□ 新增 FAQ 入库
□ 向量库索引优化
□ 用户反馈汇总→纳入下月调优计划
```

---

## 四、一句话总结

> Agent 调优不是改模型，是改 **Prompt + 工具 + 检索 + 记忆 + 护栏**。你不需要懂深度学习，你需要懂**业务逻辑 + 系统思维**。每天花 10 分钟看 Bad Case，比你集中学一周 LangChain 管用得多。
