# 模型选型评估 — 电商场景的操作流程与评估标准

> Agent 调优改的是五件套（Prompt/工具/检索/记忆/护栏）。
> 但如果模型本身能力不够，五件套改出花来也救不了。
> 这时候需要的是**模型选型评估**，而不是调优。

---

## 一、什么时候是"模型不行"？

先确认问题来源，别错怪了模型：

```
问题：Agent 回答质量差
  ↓
逐层排查：
  □ 是 Prompt 没写清楚？→ 调 Prompt
  □ 是工具没调对？→ 调工具描述
  □ 是知识库没检索到？→ 调检索
  □ 以上都调了，还是差？→ ⚠️ 可能是模型本身能力不够
```

**典型的"模型不行"信号：**

| 信号 | 描述 | 调优能不能救 |
|------|------|------------|
| 中文理解差 | 用户说"这个有白的吗"，模型理解为颜色 | ❌ 模型CJK能力天生短板 |
| 长上下文丢失 | 对话超过10轮，开始忘记前面说了什么 | ❌ 上下文窗口有效利用率不足 |
| 指令遵循弱 | 严格格式指令，模型频繁输出格式外内容 | ⭐ 部分可用Prompt强约束，但有限 |
| 工具调用混乱 | 参数传错、调了不该调的工具、忘了用工具 | ⭐ 工具描述优化可改善，上限有限 |
| 推理能力不足 | 简单的三步骤逻辑都推错 | ❌ 这是硬伤 |
| 幻觉严重 | 频繁编造不存在的信息 | ⭐ 检索增强可缓解，但本质能力问题 |

## 二、模型评估的标准框架

### 2.1 电商场景的五维评估模型

| 维度 | 权重 | 测什么 | 怎么测 |
|------|------|--------|--------|
| **中文能力** | 30% | 中文电商术语理解、多义词消歧、品牌名识别 | 100条电商中文测试集 |
| **指令遵循** | 25% | 格式遵守、JSON输出、规则执行 | 50条严格格式指令测试 |
| **工具调用** | 20% | 工具选择准确率、参数正确率 | 50条工具调用场景测试 |
| **知识检索（RAG）** | 15% | 检索结果利用、答案准确率 | 50条RAG场景测试 |
| **推理能力** | 10% | 多步推理、条件判断、归因分析 | 30条推理题（满减/退货/推荐逻辑） |

### 2.2 测试集构建

**标准测试集（100条）应该覆盖：**

```
电商智能客服测试集结构：

基础问答（30条）：
  "这件衣服什么材质？"
  "有M码吗？"
  "几天能到？"

尺码推荐（15条）：
  "我身高165体重110穿什么码？"
  "这个偏大还是偏小？"

售后退换（15条）：
  "怎么退货？"
  "七天无理由包括什么？"

价格促销（10条）：
  "能用优惠券吗？"
  "满减到什么时候？"

边��场景（20条）：
  "你帮我写个差评"（敏感）
  "这个比XX品牌好吗？"（竞品对比）
  "我就要红色没有吗？"（情绪化）
  "能再便宜50吗？"（议价）

工具调用（10条）：
  "帮我查一下订单123456"（调订单API）
  "有货吗？"（调库存API）
```

### 2.3 评分标准

| 评分 | 含义 | 标准 |
|------|------|------|
| **5分** | 完美 | 答案准确+自然+符合品牌风格 |
| **4分** | 良好 | 答案正确，小瑕疵（略啰嗦/略生硬） |
| **3分** | 及格 | 答案基本正确，但不够精准或完整 |
| **2分** | 不及格 | 答案有误或遗漏关键信息 |
| **1分** | 错误 | 完全错误或有害输出 |

**通过标准：**
- 通用场景（基础问答+尺码+售后）：平均分 ≥ 4.0
- 边界场景：平均分 ≥ 3.5
- 工具调用：准确率 ≥ 90%
- **综合：平均分 ≥ 3.8 + 无1分记录**

## 三、模型选型操作流程

### Step 1: 选候选模型（3-5个）

根据你当前场景，候选集最多5个：

```
客服/对话场景候选：
  □ DeepSeek-V3（性价比之王）
  □ 豆包2.1 Pro（256K上下文）
  □ Qwen-Max（阿里系生态）
  □ GLM-5（知识图谱强）
  □ GPT-5.5（基准对比，除非必须不选）

图像场景候选：
  □ Seedream 5.0 Pro（电商首选）
  □ FLUX.1 Pro（细节最强）
  □ SDXL + LoRA（本地，成本最低）

视频场景候选：
  □ Seedance 2.5（电商首选）
  □ Kling 3.0（中文场景好）
  □ Wan 2.1（性价比）
```

### Step 2: 构建统一测试集（半天）

为每个候选模型准备 **完全相同的测试集**。

```
关键规则：
  - 同一个Prompt模板，只换模型
  - 同一个知识库（RAG场景）
  - 同一个工具定义（工具调用场景）
  - 确保变量只有模型本身
```

### Step 3: 跑评测算分（数小时）

```
方案A：人工评分（精度高，慢）
  3个人独立打分，取平均
  → 100条 × 5个模型 × 3人 ≈ 1500条评估
  → 预计时间：1-2天

方案B：LLM-as-Judge（效率高，可靠）
  用 GPT-5.5 / Claude 4 当裁判
  → 给裁判打分规则 + 10个人工标注示例做校准
  → 速度：100条/分钟
  → 一致性（与人工对比）：通常 > 85%

方案C：混合方案（推荐）
  LLM 初筛全量 → 人工复核低分和争议项
  → 效率 + 精度平衡
```

### Step 4: 输出决策报告

```
模型评估报告（示例）：

选手：DeepSeek-V3 vs 豆包2.1 Pro vs Qwen-Max

| 维度 | DeepSeek-V3 | 豆包2.1 Pro | Qwen-Max |
|------|-------------|-------------|----------|
| 中文能力 | 4.6 | 4.8 ⭐ | 4.5 |
| 指令遵循 | 4.5 | 4.3 | 4.6 ⭐ |
| 工具调用 | 4.7 ⭐ | 4.2 | 4.4 |
| RAG利用 | 4.3 | 4.6 ⭐ | 4.2 |
| 推理能力 | 4.5 | 4.4 | 4.3 |
| **综合** | **4.52** | **4.46** | **4.40** |
| 成本(月) | 5元 ⭐ | 8元 | 12元 |

结论：
  → 推荐 DeepSeek-V3：综合最高+成本最低
  → 如果 256K 长上下文是刚需，选豆包 2.1 Pro
```

### Step 5: 灰度切换 + A/B 验证

```
不直接全量换模型，走灰度：

Day 1-3：新模型处理 10% 流量
  → 收集对比数据（旧模型 vs 新模型）
  → 看有没有降级（精度下降/延迟增加）

Day 4-7：扩大到 50%
  → 监控质检报告
  → 收集用户反馈

Day 8+：确认没问题 → 全量切换
  → 旧模型保留1个Pod做回滚
```

## 四、电商场景的模型切换成本

### 4.1 推理模型切换（低风险）

```
换模型影响范围：
  ☐ Prompt 可能需要微调（不同模型风格不同）
  ☐ 输出格式可能需要适配
  ☐ Token 计价不同

影响程度：⭐⭐ 低
切换耗时：1-3天
切换成本：主要是人工时间
风险：低（可快速回滚）
```

### 4.2 图像模型切换（中风险）

```
换模型影响范围：
  ☐ 视觉风格可能不一致（需要统一）
  ☐ LoRA 需要重新训练
  ☐ 工作流里的参数需要重新调

影响程度：⭐⭐⭐ 中
切换耗时：1-2周（含LoRA训练）
切换成本：训练费+人工
风险：中（风格统一需要时间）
```

### 4.3 视频模型切换（高风险）

```
换模型影响范围：
  ☐ 分镜参数不通用
  ☐ 视频风格不一致
  ☐ 工作流可能要大改

影响程度：⭐⭐⭐⭐ 高
切换耗时：2-4周
切换成本：重新适配工作流
风险：高（建议在项目开始时决定，中途少换）
```

## 五、什么时候该换模型

### 该换的信号

```
连续观测到的信号（至少出现3个）：

□ 测试集精度一个月内下降 > 5%
□ 用户投诉率上升 > 20%
□ 同一个Prompt，新版本模型输出劣化
□ 竞品模型评测有明显领先
□ 成本结构变化（新模型便宜50%+）
```

### 不该换的陷阱

```
❌ "换了新模型，效果应该更好吧？"
  → 不一定。新模型可能在某项强，在其他项弱。
  → 必须跑测试集，不跑不换。

❌ "大家都说这个模型好"
  → 通用场景好不等于电商场景好。
  → 电商中文+工具调用+格式遵循，你的权重和别人不一样。

❌ "先换上看看效果"
  → 没有灰度就全量换，出问题连回滚都要时间。
  → 至少留10%的旧模型流量做兜底。
```

## 六、电商场景的推荐评估周期

| 频率 | 做什么 | 产出 |
|------|--------|------|
| **每月** | 跑标准测试集，追踪精度趋势 | 精度曲线 → 发现下降趋势 |
| **每季** | 全量评测一次当前模型的综合表现 | 季度评估报告 |
| **有重大新模型发布时** | 新模型 vs 当前模型对比评测 | 切换决策报告 |
| **大促前** | 压测 + 边界场景专项测试 | 大促就绪报告 |
