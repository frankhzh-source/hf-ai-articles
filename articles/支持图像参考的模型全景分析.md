# 支持图像参考的模型全景分析（2026年7月）

## 核心发现

2025-2026年，"参考图"（reference image / image conditioning）已经成为主流模型的标配能力。但**实现方式和效果差异巨大**——不是所有"参考图"都一样的。

两条技术路线：

| 路线 | 代表模型 | 效果 | 成本 |
|:----:|:--------|:----|:----:|
| **原生解码器注入**（模型内置Image Encoder） | Seedream 4.5/4.6、DALL-E 3/4、Flux Pro 2、Imagen 4 | 🟢 极高 | 🟡 中 |
| **Adapter外挂**（IP-Adapter / InstantID / ControlNet） | SD XL/3.5、Flux Dev 等开源模型 | 🟡 中 | 🟢 低 |

---

## 一、模型逐个分析

### 1. Seedream 4.5 / 4.6（字节跳动）⭐ 你的主力

**参考图能力：当前最强之一**

| 维度 | 能力 |
|:----:|:------|
| 参考图数量 | **同时支持多达 14 张参考图** |
| 参考图类型 | 人物参考 / 服装参考 / 场景参考 / 混合 |
| 权重控制 | 0.0-1.0 可调 |
| 一致性表现 | 电商角色一致性当前综合最强 |
| 多角度生成 | 同一套参考图可生成正/侧/背不同角度，人物不漂移 |
| 价格 | $0.04/图（约 ¥0.3/张）——性价比极高 |
| API可用 | ✅ 火山引擎 API / LibTV 内置 |
| 本地部署 | ❌ 云端服务 |

**适合场景：** 电商主视觉、服装展示、多角度统一生成——你在用的，到顶了。

**参考图上传后的内部流程（LibTV 视角）：**
```
白底图 + 平铺图 + 模特图
       ↓ 通过 Seedream Image Encoder 提取特征
参考特征向量（含服装版型、印花纹理、色彩分布）
       ↓ 注入去噪网络的 Cross-Attention
保持一致性生成 不同角度/不同场景/不同动作
```

可同时传 14 张参考图，但实际场景中 3 张（白底图+平铺图+模特图）已经够用，权重设 0.8-0.9，更多参考图反而可能引入冲突。

---

### 2. Flux 2 / Flux Pro（Black Forest Labs）

**参考图能力：开源最强**

| 维度 | 能力 |
|:----:|:------|
| 参考图数量 | 1-3 张 |
| 参考图类型 | 角色参考 / 风格参考 / 上下文编辑 |
| 权重控制 | 通过 prompt 控制（无显式权重参数） |
| 一致性表现 | 角色一致性开源最强，但不如 Seedream 原生 |
| 关键架构 | 32B 参数 DiT + VLM 联动——可理解参考图内容 |
| 价格 | 本地免费 / API $0.03/图 |
| API可用 | ✅（Replicate / 自有API） |
| 本地部署 | ✅ RTX 4090 (24GB) 可跑，FP8 量化约 8 秒/图 |

**Flux 2 的参考图方式和 Seedream 不同：**

Seedream 是通过 Image Encoder 提取特征注入 Cross-Attention（专用通道）。  
Flux 2 走的是 **VLM 联动**路线——先用视觉语言模型理解参考图的内容，把"参照这张图"翻译成文本指令，再交给文生图模型。效果比 Seedream 略弱，但**可以本地部署、可以 LoRA 微调**，是开发和定制场景的首选。

---

### 3. DALL-E 4（OpenAI / GPT-5 集成）

**参考图能力：自然语言驱动**

| 维度 | 能力 |
|:----:|:------|
| 参考图数量 | 通过对话上传（无明确上限） |
| 参考图类型 | 对话式参考——上传后 GPT-5 自动理解并保留特征 |
| 权重控制 | 无显式参数（靠对话指令控制） |
| 一致性表现 | 中等——文字描述准确率 98%，但视觉一致性不如 Seedream |
| 价格 | 含 ChatGPT Plus ($20/月) 或 API $0.02-0.08/图 |
| API可用 | ✅ OpenAI API |
| 本地部署 | ❌ |

**独特之处：** DALL-E 4 不通过 Image Encoder 提取特征，而是走 GPT-5 多模态理解 → 文本描述 → 生成的路径。所以它的"参考"本质上是"看了一眼然后照着描述画"，不是"把像素特征塞进生成网络"。

优点是灵活——你可以说"脸不变，把背景换成巴黎铁塔"；缺点是不够精确——画出来的衣服版型和参考图总有点偏差。

---

### 4. Midjourney V7

**参考图能力：抽象美学参考，非精确复刻**

| 维度 | 能力 |
|:----:|:------|
| 参考图数量 | 1 张（Character Reference / Style Reference） |
| 参考图类型 | 角色参考 / 风格参考 |
| 权重控制 | **Character Reference** (`--cref` / `--sref`) 参数可调 |
| 一致性表现 | 面部一致性不错，但服装/印花等细节不精确 |
| 价格 | $10-60/月 订阅制 |
| API可用 | ⚠️ 有限 |
| 本地部署 | ❌ |

**MJ V7 的参考图通过"语义构图"和个性化 Profiles 系统实现**，本质上是把参考图的美学特征（光影风格、色调分布）而不是具体像素特征注入生成。所以：
- 风格一致性 ✅ 好（想保持某种"感觉"）
- 角色一致性 🟡 中（脸型保持，但服装细节会变）
- 商品一致性 ❌ 不好——不适合电商产品的精确复刻

---

### 5. Imagen 4（Google DeepMind）

**参考图能力：强但锁生态**

| 维度 | 能力 |
|:----:|:------|
| 参考图数量 | 多张 |
| 参考图类型 | 角色 / 风格 / 品牌参考 |
| 一致性表现 | 强——Google 内部评测率很高 |
| 价格 | 含 Google AI Pro ($19.99/月) 或 API |
| API可用 | ✅ Vertex AI API |
| 本地部署 | ❌ |

**不可忽视的选手：** Imagen 4 的参考图能力很强，在角色一致性和文字渲染上都接近 DALL-E 4 的水平。但因为发布晚、生态封闭，在电商场景的实际使用率不高。

---

### 6. Ideogram 3.0

**参考图能力：特殊场景专用**

| 维度 | 能力 |
|:----:|:------|
| 参考图数量 | 1-2 张 |
| 参考图类型 | 主要是风格/布局参考 |
| 一致性表现 | 不强——非其核心场景 |
| 核心优势 | 文字渲染准确率 > 95%，电商海报/促销图场景 |
| 价格 | 免费额度 50张/天；付费 $15/月 2500张 |
| API可用 | ✅ |
| 本地部署 | ❌ |

不适合做角色商品主视觉一致性。但做**带文字的电商海报/促销图**时最好用。

---

### 7. 开源方案组合（SD 3.5 / Flux Dev + IP-Adapter / ControlNet）

**参考图能力：灵活但需要搭**

| 维度 | 能力 |
|:----:|:------|
| 参考图数量 | 可多张 |
| 参考图类型 | 任意（取决于 Adapter 类型） |
| 权重控制 | 可调 |
| 一致性表现 | 🟡 中等——取决于底层模型和 Adapter 质量 |
| 价格 | 本地免费部署 / 云端 API $0.01-0.03/图 |
| 本地部署 | ✅ 需要 24GB+ VRAM |

如果你需要一个**不依赖任何平台、完全可控、可深度定制**的方案，这是唯一选择。但需要搭建和维护成本，对电商批量化场景来说优先级不高。

## 三、你的场景最优选型

根据你实际的使用场景（电商家居服视频生成），推荐分层策略：

```
主链路（LibTV 内）  → Seedream 4.5/4.6
    服装主视觉图、角色三视图、材质特写
    ↓
辅助链路 1（文字图） → Ideogram 3
    促销海报、标题图、带文字的封面
    ↓
辅助链路 2（私有化） → Flux 2（本地部署）
    敏感数据/客户数据/不需要联网的场景
    ↓
备选（精确指令）   → DALL-E 4
    ChatGPT 对话式微调、复杂场景描述
```

**为什么不到处用 Seedream：** 不是 Seedream 不够好，是有些场景不适用——比如促销海报带复杂文字的（Ideogram 更稳）、客户数据不能上云的（Flux 本地）、需要对话式反复修改主视觉构图的（DALL-E 4 更灵活）。

## 四、一个关键趋势：2026年是"参考图原生"的元年

2024-2025年，参考图能力是**附加功能**（IP-Adapter 等外挂方案）。
2026年，参考图能力变成**模型原生能力**（Seedream / Flux 2 / DALL-E 4 / Imagen 4 全都内置）。

这意味着：

1. **LoRA 的角色在继续收缩**——只有品牌IP精确复刻、自定义风格这两个场景还用 LoRA
2. **Adapter 方案在尴尬中间层**——不上不下，本地部署场景被 Flux 2 开源挤压，云端场景被 Seedream / DALL-E 4 挤压
3. **参考图数量和质量将继续提升**——Seedream 能传14张不是终点，明年可能到 20+ 张
