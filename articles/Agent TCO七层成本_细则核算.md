# Agent TCO 七层成本 — 逐层细则核算

> 很多人只算 Token 单价就来算 AI 的 ROI，这是错的。
> 真正的 TCO 是以下七项之和，而且 L1 通常只占 15-25%。

---

## 总公式

```
Agent TCO = Σ(L1 + L2 + L3 + L4 + L5 + L6 + L7) × 覆盖月数

每层: Lₙ = 固定成本(月/次性) + Σ(单位消耗量 × 单价)
```

---

## L1 模型调用费（15-25%）✅ 唯一被算

### 计算公式
```
L1 = Σ(月Token消耗量 × 单价)  
   = 输入Token × 输入单价 + 输出Token × 输出单价
```

### 细分项

| 子项 | 计算方式 | 典型单价 | 示例（电商Agent月消耗） |
|------|---------|---------|----------------------|
| 模型输入Token | 日均输入Token × 30天 | 豆包1.5 Pro: 0.8元/百万Token | 日均50万输入 × 30 = 1500万 → 12元 |
| 模型输出Token | 日均输出Token × 30天 | 豆包1.5 Pro: 2元/百万Token | 日均15万输出 × 30 = 450万 → 9元 |
| API调用次数附加 | 超出免费额度的API调用 | 阶梯计费 | 通常已含在Token内 |
| 模型切换成本 | 从A模型迁移到B模型的过渡期并行调用 | 按切换次数 | 每个Agent切换约500-2000元 |

### 电商场景扣费明细（一个智能客服Agent月消耗）

| 项目 | 日消耗 | 月消耗 | 月费用 |
|------|--------|--------|--------|
| 输入Token（对话历史+知识库检索） | 50万T | 1500万T | 12元 |
| 输出Token（回复生成） | 15万T | 450万T | 9元 |
| 上下文缓存命中 | 60%命中率，成本省40% | — | 抵扣约8元 |
| **月小计** | | | **约13元/Agent** |

> 注：看起来便宜，但100个Agent月消耗1300元，加上限流/重试/多模型备份，实际约3000-5000元。

## L2 算力基础设施（10-15%）❌ 常被忽略

### 计算公式
```
L2 = GPU租赁/折旧 + 带宽费用 + 电费 + 存储费用
```

### 细分项

#### 方案A：自建GPU（一次性投入+折旧）

| 项目 | 计算方式 | 示例（单台4090） |
|------|---------|-----------------|
| 显卡采购 | 硬件金额一次性 | 4090 ≈ 1.3万 |
| 月折旧（3年直线） | 采购价 / 36月 | 13000/36 ≈ 361元/月 |
| 机柜+电源 | 租赁/购买分摊 | 约200元/月 |
| 散热+运维 | 电费+人工分摊 | 约150元/月 |
| **月成本** | | **约711元/月** |

#### 方案B：云GPU租赁（即开即用）

| 平台 | 机型 | 单价 | 月估（24h/30天） |
|------|------|------|-----------------|
| 阿里云 | A100 40G | 12-18元/小时 | 8640-12960元 |
| 华为云 | Ascend 910B | 10-15元/小时 | 7200-10800元 |
| 火山引擎 | 豆包推理实例 | 3-5元/小时 | 2160-3600元 |

#### 方案C：纯API调用（推荐中小电商）

| 项目 | 计算方式 | 月估 |
|------|---------|------|
| 无需GPU，走云厂商推理API | 已含在L1 | 0元 |
| 但需要带宽保障 | 固定带宽包月 | 200-500元 |
| 存储费用（向量库/日志） | 按量 | 100-300元 |
| **月成本** | | **300-800元** |

### 电商场景说明

> 不走自建GPU的电商公司，L2主要就是带宽+向量库存储+日志存储
>
> **典型月成本**：带宽500元 + 向量库存储（Pinecone/Qdrant）300元 + 日志存储200元 = **1000元/月**

## L3 数据准备与标注（15-25%）❌ 严重被低估

### 计算公式
```
L3 = 初始数据投入(一次性) + 月均数据维护成本
    = (数据采购/清洗/标注) + (知识库构建) + (月均更新标注)
```

### 这是七层中最复杂的一层，也是最容易被低估的

### 3.1 数据来源与获取

| 数据来源 | 获取方式 | 电商场景典型成本 | 一次性还是持续 |
|---------|---------|----------------|--------------|
| 商品数据 | ERP/SCM导出 | 已有数据，清洗成本为主 | 首次最大，增量小 |
| 客服聊天记录 | 电商平台导出（千牛/抖音/京麦） | 导出免费，清洗约500-2000元 | 持续 |
| 退货/售后数据 | ERP导出 | 免费，结构化需200-500元 | 持续 |
| 直播数据 | 抖音/视频号API拉取 | API免费，清洗约1000-3000元 | 持续 |
| 竞品数据 | 爬虫采集 | 工具费500-2000元/月 | 持续 |
| 行业知识库 | 购买/整理 | 5000-20000元一次性 | 一次性 |

### 3.2 数据清洗（电商场景）

| 清洗工作 | 内容 | 耗时参考 | 人工成本（月） |
|---------|------|---------|--------------|
| 商品属性标准化 | 统一品类/颜色/尺寸/材质字段 | 100SKU=2-3天 | 约1000-2000元 |
| 客服对话清洗 | 去敏(手机/地址)→分话题→标意图 | 1000条=1天 | 约2000-5000元 |
| 图片标注 | 商品图→标签化（风格/场景/模特） | 100张=2小时 | 约500-1500元 |
| 退货原因归因 | 自由文本→结构化标签 | 500条=1天 | 约800-1500元 |

### 3.3 知识库构建（一次性大投入）

| 知识库类型 | 内容 | 构建方式 | 成本 |
|-----------|------|---------|------|
| 商品知识库 | SKU属性/卖点/规格/价格 | 手工整理+结构化 | 5000-15000元 |
| FAQ知识库 | 高频问答对 | 从历史对话抽取 | 3000-8000元 |
| 话术知识库 | 场景化话术模板 | 运营团队撰写+审核 | 5000-10000元 |
| 政策知识库 | 退换货/物流/售后规则 | 整理+结构化 | 2000-5000元 |
| 产品图库 | 商品图+场景图+模特图 | 已有资源整理 | 1000-3000元 |

### 3.4 月均持续维护

| 维护项目 | 频率 | 月工时 | 折合成本 |
|---------|------|--------|---------|
| 新SKU录入&清洗 | 每周新上架 | 约20h/周 | 约3000元 |
| 客服对话新增标注 | 每天新对话 | 约15h/周 | 约2000元 |
| 退货数据归因 | 按月 | 约8h/月 | 约500元 |
| 知识库更新 | 持续 | 约10h/月 | 约800元 |
| **月均维护小计** | | | **约6300元** |

### 3.5 电商场景典型数据规模与成本对照

| 电商规模 | SKU数 | 月对话量 | 初始数据投入 | 月维护成本 | 备注 |
|---------|-------|---------|------------|-----------|------|
| 初创（年GMV<1000万） | 50-200 | <1000条 | 5000-10000元 | 1000-2000元 | 可用AI自动标注减负 |
| 成长（年GMV 1000万-1亿） | 200-2000 | 1000-5000条 | 10000-30000元 | 3000-8000元 | 需专人维护 |
| 成熟（年GMV 1亿-10亿） | 2000-20000 | 5000-50000条 | 30000-80000元 | 8000-20000元 | 需数据团队支持 |
| 头部（年GMV 10亿+） | 20000+ | 50000+条 | 80000-200000元 | 20000-50000元 | 数据中台标配 |

### 3.6 常见低估案例

> **一家服装电商上线AI客服Agent时，只算了L1模型调用费（月2000元），结果第一周就发现精度极差——**
> 
> 因为SKU属性不全（尺码/颜色/面料描述混乱）、历史对话没清洗、退货原因没归因。
> 
> **补救成本**：临时招2个数据标注，月薪1.5万 + 知识库搭建外包2万 = **两个月花了5万**
> 
> **如果一开始就算上L3，成本预估会更准确，决策也不会走弯路。**

## L4 Agent开发与调优（10-20%）✅ 部分被算

### 计算公式
```
L4 = 框架搭建(一次性) + Prompt工程 + 测试迭代 + 版本管理
```

### 细分项

| 子项 | 计算方式 | 电商场景成本 | 说明 |
|------|---------|------------|------|
| Agent框架搭建 | 一次性 | 5000-20000元 | LangChain/CrewAI/自研 |
| Prompt编写（初版） | 每个Agent | 500-2000元/个 | 按场景复杂度 |
| Prompt调优（迭代版） | 每轮 | 300-1000元/轮 | 通常需要3-5轮 |
| 测试用例构建 | 每个场景 | 500-2000元 | 含边界Case |
| AB测试平台搭建 | 一次性 | 2000-5000元 | 对比实验环境 |
| Agent版本管理 | 月均 | 500-1000元/月 | Git+Prompt Hub |

### 电商Agent复杂度分级

| 场景 | 复杂度 | 开发工作量 | 开发成本 |
|------|-------|-----------|---------|
| 智能客服（FAQ型） | ⭐ | 2-3天 | 2000-5000元 |
| 智能客服（多轮对话） | ⭐⭐ | 5-10天 | 5000-12000元 |
| 内容生成（商品描述） | ⭐⭐ | 3-5天 | 3000-8000元 |
| 广告文案生成 | ⭐⭐⭐ | 5-8天 | 5000-10000元 |
| 直播话术助手 | ⭐⭐⭐⭐ | 10-15天 | 10000-20000元 |
| 虚拟试衣Agent | ⭐⭐⭐⭐⭐ | 20-40天 | 30000-80000元 |

### 月均TCO摊薄

```
以3个Agent（客服+内容+话术）为例：
一次性搭建 = 20000 + 8000 + 10000 = 38000元
月迭代维护 = 3个Agent × 1000元/月 = 3000元
3年摊薄 = 38000/36 + 3000 ≈ 1055 + 3000 = 4055元/月
```

## L5 系统集成（10-15%）❌ 常被忽略

### 计算公式
```
L5 = Σ(系统对接成本) + API网关建设 + 接口维护月费
```

### 细分项

| 集成对象 | 对接方式 | 一次性成本 | 月维护成本 |
|---------|---------|-----------|-----------|
| 电商平台API（淘宝/抖音/拼多多） | 开放平台SDK | 3000-8000元/平台 | 500-1500元/月 |
| ERP/OMS系统 | API/DB直连 | 5000-15000元 | 1000-3000元/月 |
| WMS仓储系统 | API对接 | 3000-8000元 | 500-1500元/月 |
| 客服系统（千牛/抖音IM） | 开放接口 | 2000-5000元 | 500-1000元/月 |
| 数据看板（BI/飞书） | API/Webhook | 2000-5000元 | 300-800元/月 |
| API网关（统一入口） | 一次性建设 | 5000-10000元 | 800-2000元/月 |

### 电商企业典型成本

| 规模 | 对接平台数 | 一次性集成成本 | 月维护 |
|------|-----------|--------------|--------|
| 单平台店铺 | 1-2个 | 8000-15000元 | 1000-2000元 |
| 多平台铺货 | 3-5个 | 20000-40000元 | 3000-5000元 |
| 品牌全渠道 | 5-10个 | 40000-80000元 | 5000-10000元 |

## L6 运维监控（5-10%）❌ 几乎未算

### 计算公式
```
L6 = 监控系统 + 告警通道 + 日志管理 + Agent健康检查
```

### 细分项

| 项目 | 计算方式 | 月估成本 |
|------|---------|---------|
| Agent运行监控 | 自建/第三方（LangSmith/Weights&Biases） | 500-2000元/月 |
| Token消耗监控 | 仪表盘+告警 | 200-500元/月 |
| Agent精度监控 | 周期性抽样评估+自动化测试 | 500-1500元/月 |
| 日志采集与存储 | 按量计费 | 300-1000元/月 |
| 异常告警通道 | 飞书/钉钉/企微Webhook | 100-300元/月 |
| 故障排除 | 工程师应急响应（人时折算） | 500-2000元/月 |
| **月小计** | | **2100-7300元/月** |

### 典型的"Agent崩了没人知道"场景

```
一个智能客服Agent在凌晨2:00因API限流返回值异常，自动回复了"系统繁忙"
→ 早上8点才发现，期间产生了1200条无效回复
→ 损失：客户体验差评 + 可能漏单

如果装了监控+L6成本约200元/月，凌晨2:01就收到告警并自动切换备用模型
```

## L7 安全合规（5-10%）❌ 几乎未算

### 计算公式
```
L7 = 权限体系 + 审计日志 + 隐私合规 + 数据安全
```

### 细分项

| 项目 | 计算方式 | 月估成本 |
|------|---------|---------|
| 权限体系搭建 | 一次性 | 3000-8000元 |
| 审计日志 | Agent行为的完整记录 | 500-1500元/月 |
| 数据脱敏 | 手机号/地址/支付信息等 | 500-2000元/月 |
| 合规审核（电商特殊） | 广告法/虚假宣传/价格合规 | 1000-3000元/月 |
| 等保/安全认证 | 按年度 | 摊薄1000-3000元/月 |
| 应急响应 | 安全事故处理 | 200-500元/月（摊薄） |

### 电商场景特殊风险

| 风险点 | 说明 | 潜在损失 |
|-------|------|---------|
| Agent生成虚假优惠信息 | 模型幻觉说"满100减50"实际无此活动 | 客诉+赔偿+工商处罚 |
| Agent泄露客户信息 | 对话中透露手机号/地址给第三方 | 合规处罚+品牌声誉 |
| Agent违反广告法 | "最好""第一""绝对"等极限词 | 工商罚款20-100万 |
| 价格错误 | 模型算错折扣导致超低价出售 | 按GMV损失，典型5000-50000元 |

## 全层汇总：一个中等规模电商的月TCO

假设场景：年GMV 3000万，抖音+淘宝双平台，3个Agent（客服+内容+话术）

| 层级 | 成本项 | 一次性投入 | 月成本 |
|------|--------|-----------|--------|
| L1 | 模型调用费 | — | 3,000-5,000 |
| L2 | 算力基础设施 | — | 800-1,500 |
| L3 | 数据准备与标注 | 30,000-50,000 | 6,000-10,000 |
| L4 | Agent开发与调优 | 38,000-50,000 | 4,000-7,000 |
| L5 | 系统集成 | 20,000-40,000 | 3,000-5,000 |
| L6 | 运维监控 | 3,000-5,000 | 2,000-4,000 |
| L7 | 安全合规 | 10,000-20,000 | 2,000-5,000 |
| **合计** | | **104,000-165,000** | **20,800-37,500/月** |

```
月均TCO（含一次性摊薄/24个月）：
= (104,000-165,000)/24 + (20,800-37,500)
= 4,333-6,875 + 20,800-37,500
= 25,133-44,375元/月

其中 L3 数据层：占28-40%，是最重的单项
```

---

## 关键结论

1. **L1只占15-25%** — 只算Token单价来计算ROI，误差可达4倍以上
2. **L3是最重的单项** — 数据准备与标注占15-25%，且初始投入大（一次性3-5万）
3. **L3+L5+L6+L7合计占45-65%** — 这四层是"隐藏冰山"，几乎无人计算
4. **规模效应在L2/L5/L6最明显** — 从1个Agent扩到100个，这些成本不会线性增长
